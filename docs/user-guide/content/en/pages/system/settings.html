Settings

Page · System · 5 min

Central configuration hub for all AlphaLoop preferences — language, profile, exchange, risk control, notifications, API keys, MCP server, data vacuum, and advanced operations.

TL;DR

Settings is organized into 7 tabs: General (language toggle + profile), Trading (exchange config + risk parameters), Notifications (Telegram integration + event toggles), API (third-party API keys), Services (MCP server status + audit log), Data (data vacuum/archival), and Advanced (data export + account deletion). Each tab switch is animated with a sliding green indicator. All settings support local modification and cloud persistence.

Who uses it & when

Every user at some point. First launch: configure language, exchange, and API keys. Risk parameters are set before a strategy goes live. Notifications are configured when Telegram alerts are needed. Data vacuum runs when storage is running low. Advanced operations are reserved for administrators.

Page Anatomy

  1. ① Tab bar — 7 section tabs: General, Trading, Notifications, API, Services, Data, Advanced. Active tab has a green sliding indicator with glow shadow.
  2. ② General settings — Language toggle (简体中文 / English, instant effect), profile (username, email, role, timezone, 2FA status).
  3. ③ Exchange configuration — Exchange selector (Binance/OKX/Bybit), trading mode (spot/futures), dry-run toggle, API Key/Secret fields.
  4. ④ Auto-save — Settings are persisted to the backend with a 2-second debounce. Local defaults are used offline without blocking the user.
  5. ⑤ Advanced operations — Data export (strategies + trade history) and account deletion. Deletion requires a confirmation dialog.

Key Settings Explained

  • Language toggle — Switch between Simplified Chinese and English. All pages respond instantly without restart.
  • Dry Run (Paper Trading) — When enabled, the exchange API is used for market data only; no real orders are placed. Essential during strategy validation.
  • Risk parameters — Max single loss (default 5%), max drawdown (15%), daily drawdown limit (8%), auto-pause. The system automatically halts new entries when thresholds are breached.
  • Notification configuration — Telegram Bot Token and Chat ID for trade execution, risk events, daily summary, and system alert notifications.
  • MCP service — View MCP server status (bind address, request count), audit log (response status codes and latency per tool call), and rotate access tokens.
  • Data vacuum — Scan and archive expired signal data to free up storage. History shows scanned and archived counts per run.

Common Actions, Step by Step

  1. First use: open the General tab to choose a language and fill in your profile. Switch to the Trading tab to configure your exchange and API keys — confirm dry-run mode is enabled.
  2. Configure risk controls: set max single loss and max drawdown in the Trading tab, and enable auto-pause.
  3. Connect Telegram: enter your Bot Token and Chat ID in the Notifications tab, then toggle the event types you want to receive.
  4. Configure AI services: enter OpenAI or Anthropic API keys in the API tab. Check MCP audit logs in the Services tab.
  5. Run data vacuum: when storage is low, click "Run Vacuum" in the Data tab. Check the history for results.
  6. Reset risk defaults: click "Reset Defaults" at the bottom of the risk parameters section to restore factory values.

Behind the Scenes

Settings state is managed by SettingsState (a singleton initialized with dependency injection). On first launch, it fetches saved settings from /auth/settings on the backend, overriding local defaults. Every modification triggers scheduleSave() — a 2-second debounce window pushes changes to /auth/settings. Language switching is global through the L10n system; views re-render via .id(settings.language). When offline, all settings retain their local defaults without blocking the user.
