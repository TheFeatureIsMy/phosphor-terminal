Data Source Management

Page · System · 5 min

Central management for every data pipeline feeding into AlphaLoop — market kline, order book, funding rates, on-chain data, news, and social sentiment — with latency, freshness monitoring, enable/disable toggles, and connection testing.

TL;DR

Data Source Management is the control center for all of AlphaLoop's data pipelines. Sources are organized by category (Kline, Order Book, Funding Rate, Open Interest, News, Whale, On-Chain, Research, Social Sentiment), with each source displayed as a card showing latency, freshness, last fetch time, and status. Summary cards at the top show totals for all sources, active count, errors, and categories. You can test individual connections, batch-test all active sources, or toggle sources on and off.

Who uses it & when

Anyone configuring a new exchange or data provider who needs to verify connectivity. Also useful when a strategy's signals look off — check whether the relevant data source has expired or entered an error state. Good for post-migration verification by batch-testing all active sources.

Page Anatomy

  1. ① Summary cards — Total source count, active, errors, and category count. Investigate when the error count is above zero.
  2. ② Category filter chips — Filter sources by category: Kline, Order Book, Funding Rate, Open Interest, News, Whale, On-Chain, Research, Social Sentiment.
  3. ③ Data source cards — Each card shows name, status indicator, provider, latency (ms), freshness (fresh/stale/expired), and last fetch time. Action buttons for "Test" and "Enable/Disable" at the bottom.
  4. ④ Status warning banner — Automatically shown when any data source has an error or degraded state, listing reason codes for quick diagnosis.
  5. ⑤ Batch test — "Test All Connections" button in the page header runs connectivity checks on every active source sequentially.

Reading the Key Metrics

  • Latency — Round-trip time for data requests. Green < 100ms, yellow 100-300ms, orange > 300ms. Consistently high latency may indicate network issues or provider problems.
  • Freshness — Time since the last successful data fetch. Green = fresh (timely data), yellow = stale (slight delay), red = expired (data is unreliable). Expired data can distort strategy signals.
  • Status — Active (collecting normally), Disabled (manually turned off), Error (connection failed), Rate Limited (frequency cap hit). Error and rate-limited states need human intervention.
  • Last Fetch — Relative time display (just now / 5 min ago / 2 hr ago) helps you judge whether data is still flowing.

Common Actions, Step by Step

  1. After adding a new data source, find its card under the relevant category and click "Test" to verify the connection.
  2. If the test fails, check whether the API key is valid, the network is reachable, and the provider has rate limits.
  3. Click "Disable" on sources you do not currently need to avoid wasting resources on unnecessary requests.
  4. For bulk operations, click "Test All Connections" to verify all active sources at once.
  5. Read the reason codes on error-state sources (e.g. rate_limit_exceeded) and take the suggested action.

Behind the Scenes

Page data is fetched via the ViewModel's load() method, calling the backend API to return all data source states and metrics. Each "Test" operation triggers an independent connectivity check, with results updating the card in real time. "Test All Connections" iterates through active sources sequentially. The enable/disable toggle updates the source configuration on the backend. Category filtering is local and does not trigger new network requests.
