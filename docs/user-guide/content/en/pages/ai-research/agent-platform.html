Agent Platform

Page · AI Research · 5 min

Configure, monitor, and manage your AI trading agents — each one is an independent research analyst that scans markets and produces signals on a schedule.

TL;DR

The Agent Platform is your "control room" for AI agents. Each agent is an independently configurable analyst role with its own LLM provider, research preferences, and output format. From here you can see every agent's running status, recent signal statistics, and detailed performance. Agents are like your research team — assign them different assets to track, adjust analysis depth, and even choose different AI model backends.

Who uses it & when

Advanced traders and quant developers managing multiple strategies. Use the Agent Platform when you want AI to continuously track multiple assets and produce regular research reports and signals. Also useful after deploying a new agent to check whether it's working correctly and what signal quality looks like.

Page Anatomy

  1. ① Header Bar — Page title and active agent count. Updates in real time as you add or remove agents.
  2. ② Agent Card Grid — Each agent is a card showing its name, LLM provider, status indicator (green=online, orange=warning, red=faulted), weekly signal count, and average signal quality score. Tap a card to open its detail panel.
  3. ③ Agent Detail Panel — View and edit configuration: LLM provider, research focus (e.g., Technical+On-Chain, Sentiment only), output language. Recent signals from this agent appear below.

Reading the Key Metrics

  • Average Score — The agent's signals' average score in the fusion system. Consistently above 70% means the agent performs well; consistently below 40% may need configuration adjustments or data source checks.
  • Weekly Signal Count — Quantity alone isn't quality. An agent producing 2 high-precision signals per week is often more valuable than one producing 20 low-quality signals. Read this together with the average score.
  • Status Dot — Green = running normally; Orange = running but some data sources are degraded; Red = agent faulted (usually an LLM API call failure), check API key or network connectivity.

Common Actions, Step by Step

  1. Browse the agent card grid to see all agents' running status and performance at a glance.
  2. Tap an agent card to open its detail panel for configuration and recent signals.
  3. Edit LLM provider (e.g., switch from OpenAI to Anthropic) or adjust research focus in the detail panel.
  4. Return to the grid and tap "Add Agent" to create a new AI agent.
  5. Configure the new agent's name, research focus, output language, and LLM provider.
  6. Check back after a while to review signal quality — if the average score stays low, try a different research focus or LLM provider.

Behind the Scenes

The Agent Platform manages each agent's configuration and lifecycle through tradingagents_adapter.py. Each agent is essentially a TradingAgents instance with its own LLM provider and preference parameters. Signals produced by agents are scored through signal_scoring.py's five-dimension system (verifiability, evidence, specificity, novelty, risk awareness — weighted into a 0-5 score) before entering the Signal Center. Agent configurations persist in the database and load on each startup.
