Execution Center

Page · Execution · 5 min

The Execution Center is the master control panel of AlphaLoop — see all active strategy sessions at a glance, toggle between dry-run and live modes, and hit the emergency stop when things go wrong.

TL;DR

The Execution Center aggregates every running Freqtrade session into one view. Five summary cards at the top show running sessions, open positions, pending orders, Freqtrade heartbeat status, and execution latency. Below is a detailed session list with mode badges (dry-run / live / paper), strategy names, and position counts. The red emergency stop button in the top right lets you halt all trading instantly when needed.

Who uses it & when

Anyone who needs to know "is my strategy running, and is it healthy?" should check the Execution Center first. Open it every morning before market open, after deploying a new strategy, or when an alert fires. If you only look at one page to understand system health, this is the one.

Page Anatomy

  1. ① Status Banner — Appears when Freqtrade heartbeat is missing or the exchange connection is broken. Displays specific reason_codes for diagnosis.
  2. ② Summary Cards — Running sessions, open positions, pending orders, Freqtrade heartbeat (Healthy/Degraded/Down), and execution latency (<100ms green / <300ms yellow / above orange). The emergency stop button requires a confirmation dialog.
  3. ③ Session List — One row per Freqtrade session showing strategy name, trading pair, mode badge (dryrun=yellow / live=red / paper=green), open positions, and pending orders. A yellow triangle indicates reason_codes that need attention.
  4. ④ Color Legend — Green=running / Yellow=paused / Gray=stopped / Red=error.

Reading the Key Metrics

  • Running Sessions — How many strategy sessions are currently active. If this is lower than expected, check whether a session stopped unexpectedly.
  • Freqtrade Heartbeat — Healthy means the Freqtrade engine is responding normally; Degraded means partial functionality with elevated latency; Down means the connection is lost and you should check the Freqtrade service.
  • Execution Latency — Round-trip time from command issuance to exchange confirmation. Under 100ms is normal; above 300ms indicates network or exchange load issues that may affect fill quality.
  • Emergency Stop — Sends a termination signal to all active sessions. Only use when the system is out of control or a strategy is behaving unexpectedly. Sessions must be manually restarted afterward.

Common Actions, Step by Step

  1. View session details — Click any session row to navigate to its Orders & Positions page for the full trade log.
  2. Emergency stop all trading — Click the red "Emergency Stop" button in the top right → confirm in the dialog → the system broadcasts a stop signal to all Freqtrade sessions → the page refreshes automatically.
  3. Switch mode — The Execution Center does not provide mode switching. Start new sessions in dry-run, paper, live_small, or live mode from the strategy management page.
  4. Refresh state — Data loads automatically when the page opens. To manually refresh, navigate away and back.

Behind the Scenes

The Execution Center fetches data from the BFF endpoint /api/execution/center. The backend first tries the Redis cache for session states (latency-critical), falls back to database queries on the Freqtrade sessions and heartbeat tables, and finally returns mock data if the database is empty. Session states are periodically synchronized via the Freqtrade REST API through freqtrade_client.py, while historical trade data is read directly from the Freqtrade SQLite database via freqtrade_db.py. The emergency stop calls /api/execution/emergency-stop, which broadcasts a stop command to all running Freqtrade sessions.
