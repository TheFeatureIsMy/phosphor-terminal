Orders & Positions

Page · Execution · 5 min

Track every order through its entire lifecycle — from placement to fill — and monitor all open positions with real-time P&L.

TL;DR

The Orders & Positions page has two tabs: Orders lists every historical order (side, symbol, quantity, price, status), and Positions shows all currently open positions (entry price, current price, unrealized P&L, stop loss). Connection status indicators for Exchange and Freqtrade sit at the top, with warning banners when something is wrong. The page also serves as the detail view for the execution records — clicking "Orders" on a run row opens this data in a sheet.

Who uses it & when

Traders check this page daily to review open positions and verify order fills. Use it when you need to confirm whether an order went through, check floating P&L, or see if a stop loss has been triggered. It is also the detail drill-down target from the Execution Records view.

Page Anatomy

  1. ① Connection Status — Real-time health indicators for the exchange and Freqtrade connections. Green=normal, Red=error. A banner expands with reason_codes when something is wrong.
  2. ② Tab Switcher — Orders and Positions tabs each show a count badge. The active tab is highlighted in green.
  3. ③ Orders/Positions List — Each row is one record. Order rows show direction (BUY/SELL), symbol, order type, quantity, price, status badge, and exchange order ID. Position rows additionally show entry price, current price, unrealized P&L, and stop loss.
  4. ④ Color Legend — Status colors, side colors, and P&L colors are explained at the bottom.

Reading the Key Metrics

  • Order Status — filled means fully executed; pending/open means waiting on the order book; partially_filled means only part of the order was executed; cancelled or rejected needs investigation.
  • Unrealized P&L — Position rows show floating profit/loss as both an absolute value and a percentage. Green for profit, red for loss. A large unrealized loss means you should check whether your stop loss is appropriate or whether manual intervention is needed.
  • Entry Price vs Current Price — The gap directly determines your P&L. If the current price is approaching your stop loss level, the system will auto-close the position — but it is better to act before it does.
  • reason_codes — A yellow triangle on any row indicates additional metadata is available (risk flags, source trace info, etc.). Hover to see details.

Common Actions, Step by Step

  1. View order details — Click the "Orders" button on a run row in the Execution Records page to open a detail sheet with the order list, ledger events, and signal trace chain.
  2. Filter by mode or status — In the Execution Records page, use the filter bar to narrow runs by mode (backtest / dryrun / live_small / live) or status (running / completed / stopped / failed).
  3. View the trace chain — In the detail sheet's "Trace" tab, follow the six-step chain: Signal → Agent → Strategy → TradeIntent → RiskDecision → Order.
  4. Track ledger events — The "Events" tab in the detail sheet shows fund movements for each trade (trade_pnl, fee, funding, deposit, withdrawal) with a timeline and balance changes.

Behind the Scenes

Orders and positions data comes from the BFF endpoint /api/execution/orders-positions. The Freqtrade REST API provides real-time pending orders and open positions, while the historical order details are read directly from the Freqtrade SQLite database via freqtrade_db.py. These two data sources are reconciled by the Reconciliation Bus — any discrepancies between local records and the exchange state are flagged in reason_codes. The trace chain data is rendered by the TradeSourceTrace component, which integrates signal source, strategy DSL version, runtime snapshot, and risk decision information into a single vertical flow.
