Backtest & Simulation

Page · Strategy · 5 min

Validate your strategy against historical data before committing real capital, or observe its real-time behavior in a paper trading environment.

TL;DR

The BacktestLab page uses a single-column data-terminal layout (redesigned 2026-07-01). A 48pt top bar houses the RunSwitcher + Backtest/Dryrun segmented control + New Run button + Compare button (appears when 2+ runs are checked). Below sits the 360pt EquityCurveHero (gradient-filled equity curve + compare overlay), a collapsible StrategyContextStrip (strategy + risk + promotion gate), a 2x4 MetricsGrid (8 KPI cards), and a collapsible TradeListTable (20-row pagination). Right-side drawers: NewRunDrawer (config form) and HistoryDrawer (run list + compare checkboxes), managed by DrawerContainer. The old three-column layout has been replaced.

Who uses it & when

Every strategy author. Backtesting is mandatory before going live — run it after every DSL or canvas change. The BacktestLab page (from the sidebar) offers the full backtest engine. Paper trading (simulation) is the final step after satisfactory backtest results, used before going live.

Page Anatomy

  1. ① Top Bar (48pt) — The RunSwitcher button on the left shows the currently selected run (number, symbol, date); click to open the HistoryDrawer. The Backtest/Dryrun segmented control sits in the center. The "New Run" button on the right opens the NewRunDrawer. When 2+ runs are checked in history, a "Compare" button appears to toggle equity curve overlay mode.
  2. ② EquityCurveHero (360pt) — The hero equity curve visualization. In single-run mode, displays a gradient-filled curve with three inline labels (Total Return / Max Drawdown / Peak). In compare mode, overlays multiple curves with distinct colors and a legend. Shows an empty area when no data is available.
  3. ③ StrategyContextStrip — A collapsible strategy context bar. Collapsed state shows a one-line summary: "Strategy #ID · N warnings · gate status". Expanded view shows strategy details, risk warnings, and promotion gate status.
  4. ④ MetricsGrid (2x4) — A 2-row, 4-column grid of 8 KPI cards: Total Return, Max Drawdown, Sharpe Ratio, Win Rate, Profit Factor, Total Trades, Filled, and Avg Duration. Uses monospaced digits with green for positive and red for negative values.
  5. ⑤ TradeListTable — A compact, collapsible trade detail table. Displays the first 20 rows by default; click "Show all" to expand. Columns: number, time, side, entry, exit, P&L, duration. Shows "No trades" when empty.
  6. ⑥ Drawer Panels — Slide-in panels managed by DrawerContainer. NewRunDrawer contains the new run config form (symbol, timeframe, strategy ID). HistoryDrawer lists historical runs with compare checkboxes and search support.

Section Reference

  1. RunSwitcher — Top bar button showing the current run's number, symbol, and date. Click to open the HistoryDrawer for switching runs. Shows "Select run" placeholder when no run is selected.
  2. EquityCurveHero — 360pt hero equity curve chart. Dark terminal style with surfaceHover.opacity(0.35) background and border. Gradient fill in single mode, multi-color overlay in compare mode. Three inline metrics (Total Return, Max Drawdown, Peak).
  3. StrategyContextStrip — Collapsible strategy context area. One-line summary when collapsed; strategy details, risk warnings, and promotion gate status when expanded.
  4. MetricsGrid — 2x4 grid of 8 core backtest KPIs. Monospaced digits, color-coded by sign.
  5. TradeListTable — Collapsible trade list with 20-row pagination and fixed headers.
  6. NewRunDrawer — Right-side drawer config form. Configure symbol, timeframe, and strategy ID, then start a backtest or dryrun.
  7. HistoryDrawer — Right-side drawer listing historical runs with search and compare checkboxes. Click a row to switch focus.

Reading the Key Metrics

  • Total Return — The percentage profit over the backtest period. Green for positive, red for negative. Always evaluate alongside Max Drawdown: 40% return with 5% drawdown is much better than 40% with 25%.
  • Sharpe Ratio — Risk-adjusted return. >=1.5 is excellent (green), 1.0-1.5 is acceptable (yellow), <1.0 is low. It tells you how much excess return you earn per unit of volatility.
  • Max Drawdown — The largest peak-to-trough decline during the backtest. Always shown in red. For crypto strategies, 20-40% drawdowns are common but depend on the strategy type.
  • Win Rate — The proportion of profitable trades. >=50% is green. But higher isn't always better — a high win rate with a low profit factor may underperform a lower win rate with a high profit factor.
  • Profit Factor — Gross profit / gross loss. >=1.5 is green. If the profit factor is high, the strategy can still be profitable even with a 40% win rate.

Common Actions, Step by Step

  1. Run a backtest — Open the BacktestLab page from the sidebar. Click "New Run" in the top bar to open the NewRunDrawer. Fill in the symbol, timeframe, and strategy ID, then click "Backtest". The new run appears in the HistoryDrawer.
  2. Review results — Select a completed run. EquityCurveHero shows the equity curve, MetricsGrid displays 8 KPIs, TradeListTable shows trade details. Collapse/expand StrategyContextStrip for strategy context.
  3. Compare multiple runs — Check 2-3 runs in the HistoryDrawer. The "Compare" button appears in the top bar. Click to enter compare mode, where EquityCurveHero overlays multiple curves.
  4. Switch to paper trading — Click the "Dryrun" tab in the top bar to switch to paper trading mode. Configure a dryrun the same way as a backtest.
  5. Search history — Use the search box in the HistoryDrawer to filter historical runs by symbol.

Behind the Scenes

Backtesting runs asynchronously via the Command Bus. After submission, the frontend polls for status updates. The page state machine transitions through idle → configuring → running → completed / failed. In compare mode, the frontend collects selected runIds and fetches comparison data from the backend. All data panels use the dark terminal style: surfaceHover.opacity(0.35) background with border, monospaced PulseFonts.tabular digits, no glassmorphism or serif italics.
