Strategy Canvas

Page · Strategy · 5 min

Visually build your trading strategy by dragging and dropping nodes — create professional-grade quant logic without writing a single line of code.

TL;DR

The Strategy Canvas is a visual DAG (directed acyclic graph) editor built on React Flow, embedded inside the macOS app via WKWebView. A node palette on the left lists all available components (Signal Input, Indicator Condition, Filter, Position Sizing, Risk Policy, Execution Output, MTF Guard, etc.). Drag nodes onto the canvas and connect them to build your strategy. A DSL preview panel on the right shows the generated JSON in real time. The validation status bar at the bottom tells you whether the strategy is valid before saving.

Who uses it & when

Perfect for visual thinkers — when writing DSL JSON by hand feels tedious, the canvas makes strategy construction intuitive. Open it from the "Canvas" tab inside a strategy's detail page, or from the sidebar as a standalone page where you can switch between strategies. The Template Library provides preset blueprints (RSI Mean Reversion, Bollinger Breakout, MACD Trend Following, etc.) that load onto the canvas as a starting point.

Page Anatomy

  1. ① Node palette (left) — Lists all available components: Signal Input, Indicator Condition, Filter, Position Sizing, Risk Policy, Execution Output, MTF Guard, Structure Defense. Click a node to add it to the canvas, or drag and drop. Filter by category or search by name. Favorited nodes appear at the top.
  2. ② Canvas area (center) — An infinite canvas built on React Flow. Nodes appear as colored cards with circular ports on the left (input) and right (output). Drag from an output port to an input port to create an edge. Supports zoom, pan, multi-select (Cmd+click), and keyboard delete. A minimap in the bottom-right corner helps with navigation.
  3. ③ Node config panel (right) — Opens when you click a node on the canvas. Shows core parameters (e.g. RSI period, threshold), port connection status, and advanced options (name, notes, execution condition). Changes take effect immediately.
  4. ④ Status bar (bottom) — Shows validation status (Passed / Failed / Not Validated) and canvas statistics (node count, edge count). The "DSL Preview" toggle button is on the right.
  5. ⑤ DSL Preview panel (optional) — Slides in from the right when you toggle "DSL Preview". Shows the real-time JSON output of your canvas graph. Supports copy to clipboard.

Reading the Key Metrics

  • Validation status — Green "Validation Passed" means the canvas graph forms a complete, compilable strategy. Red "Validation Failed" with an error count means you need to fix issues before saving.
  • Node and edge count — Shown in the status bar. More complex strategies have more nodes, but good strategies tend to be "as simple as possible."
  • Port status — In the config panel, connected ports show a green dot and the peer node name; unconnected ports are gray; required ports have a red asterisk.

Common Actions, Step by Step

  1. Add a node — In the left palette, click (or drag) a node type such as "Indicator Condition". It appears in the center of the canvas.
  2. Connect nodes — Drag from a node's output port (right-side circle) to another node's input port (left-side circle). Edges use a smooth step path style.
  3. Configure a node — Click a node on the canvas; the right panel opens automatically. Adjust parameters (e.g. RSI period=14, threshold <30). Changes reflect immediately in the DSL preview.
  4. Validate and save — Click "Validate" in the top bar to check strategy legality. Once passed, click "Save" to persist the canvas as a new strategy version.
  5. Load a template — Click "Templates" and pick from preset blueprints (e.g. "RSI Mean Reversion"). The template's nodes and edges load onto the canvas automatically.
  6. View DSL code — Click "DSL Preview" in the status bar. The right panel shows the generated JSON. Click the copy icon to copy it to your clipboard.

Behind the Scenes

The canvas is a React application embedded in the macOS app via WKWebView. It uses @xyflow/react for the DAG editor. Node and edge changes are sent to the Swift side through CanvasBridge (a WKScriptMessageHandler). The graph-to-DSL conversion is done client-side by graphToDsl(), which maps nodes and edges to the StrategyRuleDSL v2.5 JSON format. Saving calls APICanvas.save() to persist the graph state and APIStrategiesV2.saveVersion() to create a new strategy version. Version history is viewable in the "Versions" tab of the strategy detail page.
