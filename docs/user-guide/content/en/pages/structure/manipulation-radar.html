Manipulation Radar

Page · Structure · 6 min

A nine-section narrative flow centered on a single suspected manipulation case. Surfaces multi-layer evidence, lifecycle stage, conservative/aggressive defense options, and the real impact on currently enabled strategies — while honestly flagging missing data and uncertainty.

TL;DR

The Manipulation Radar is not a verdict system — it is a statistical inference engine built on multi-source data signals. The top of the page is a horizontally scrolling Active Cases Strip (Hero Strip); click any card to switch the focused case. Below, the page unfolds the focused case through nine sections: Verdict → Lifecycle Timeline → 5-Layer Evidence Matrix (price/volume, orderbook liquidity, on-chain, social/news, cross-market) → Whale & Concentration → Cross-Market Pressure → Social Acceleration → Defense & Strategy Impact (conservative/aggressive side by side) → Alerts + Similar Historical Cases.

Who uses it & when

Every trader — especially those involved in altcoins and low-liquidity perpetuals. Two typical user profiles:

  • Conservative — quickly verify "is this symbol being manipulated?" before trading. If the page shows "likely distribution stage" or "likely collapse stage", stay away.
  • Aggressive — understand the lifecycle of manipulation and ride the early stages (accumulate / markup). The Defense panel always presents both stances so you can compare.

Page Anatomy (Nine Sections)

  1. Masthead — Title + system-level disclaimer: "Manipulation radar is a statistical inference system; it surfaces evidence-based suspicions, not verdicts."
  2. §0 Active Cases Strip — Horizontally scrolling thumbnail cards for currently active cases. Each card: symbol · M-type · current stage · confidence. Clicking a card switches focus across all sections below.
  3. §1 Verdict Panel — Conclusion for the focused case: M-type chip · lifecycle stage progress · confidence ring · risk level · data completeness (N/5 layers). All copy uses probabilistic prefixes ("Likely" / "Evidence consistent with") — never a definitive judgment.
  4. §2 Lifecycle Timeline — Horizontal timeline of 5 stage nodes (SUSPECTED → ACCUMULATE → MARKUP → DISTRIBUTE → COLLAPSE). The current node glows, past nodes are filled, future nodes are dashed outlines. Click a node to expand: entry time + confidence at that moment + key feature snapshot.
  5. §3 Evidence Matrix — 5 rows × n columns: Layer A Price/Volume · Layer B Orderbook Liquidity · Layer C On-Chain · Layer D Social & News · Layer E Cross-Market. Each row shows its composite anomaly score + a data_quality badge (0-1). Layers with data_quality < 0.3 are marked "Data unavailable" and excluded from the verdict. Expand any row to see individual features + z-scores + historical percentiles.
  6. §4 Whale & Concentration (Layer C detail) — Top-10 holder concentration trend + recent 24h large transfers + exchange net inflow.
  7. §5 Cross-Market Pressure (Layer E detail) — Funding rate z-score + spot-perpetual basis + open interest change + long/short ratio + 24h liquidation volume.
  8. §6 Social Acceleration (Layer D detail, may be missing) — Twitter / Telegram mention velocity + KOL mention timeline + sentiment extremity. If the layer is unavailable, the entire section honestly displays "Data unavailable".
  9. §7 Defense & Strategy Impact — Side-by-side columns: Conservative recommendation (blue, e.g. "Exit now") and Aggressive recommendation (amber, e.g. "Reduce or short"). Below: affected symbols chip list + strategy-impact table (each enabled strategy shows whether its ManipulationFilter is enabled / would block this case / its reason codes) + a button jumping to risk configuration.
  10. §8 Alerts + Similar Historical Cases — Left half: real-time alert timeline for this case (stage transitions, anomaly spikes), pushed via WebSocket. Right half: top-N historical completed cases matched by evidence-vector similarity, including their outcomes (peak change / collapse depth / duration) for calibration.

Reading the Key Metrics

  • Confidence — 0-1, the strength with which evidence supports the verdict. Capped by data completeness: missing layers lower the ceiling (max_confidence = min(completeness × 1.2, 1.0)).
  • Data completeness — How many layers (out of 5) supply trustworthy data. 3/5 means only 3 layers contributed; the verdict is grounded in those 3 and never "imagines" missing ones.
  • data_quality badge — Per-layer reliability marker. >0.7 reliable, 0.3-0.7 reference only, <0.3 treated as unavailable.
  • Risk level — low / medium / high / extreme, derived from composite anomaly score combined with current lifecycle stage. Distribution / collapse stages naturally escalate the level.
  • Dual-profile recommendations — Conservative gives defensive guidance ("avoid / cautious entry / exit now"); Aggressive gives same-evidence trend-following guidance ("small ambush / ride / reverse short"). Both are shown side by side — the user chooses a stance based on their own risk profile and position context; the system does not decide for them.
  • Strategy impact — Lists currently enabled strategies whose symbols match this case. Each row notes whether the strategy's manipulation_score_filter rule is enabled, whether it would block, and the block reason codes.

Common Actions, Step by Step

  1. Open the Manipulation Radar page; the Hero Strip defaults to focusing the first active case.
  2. Click any card in the Strip to switch focus — all sections below load the new case in parallel via three endpoints (detail / strategy-impact / similar).
  3. Read the §1 Verdict panel; pay attention to the confidence ring and "N/5 layers". If completeness is only 2/5, treat the verdict as an "early signal" rather than a firm conclusion.
  4. Follow §2 Lifecycle Timeline to understand which stage the case is in. Click any node for entry time and feature snapshot.
  5. In §3 Evidence Matrix find the highest-scoring layer; expand it to see the specific features (e.g. volume_zscore = 2.4, top_10_concentration = 0.62).
  6. Jump to §7 Defense & Strategy Impact; compare the two stances and decide. If a strategy's ManipulationFilter is shown as disabled, click "Open risk config" to inspect in Risk Center.
  7. Read §8 similar historical cases — how deep did similar manipulations finally collapse? How long did they last? Calibrate your expectations accordingly.

Behind the Scenes

Page data comes from the /api/v2/manipulation/* endpoint family: /radar (overview + active list), /cases/{id} (case v2 with per-layer evidence + data_quality + dual-profile signal), /cases/{id}/strategy-impact (strategy scan), /cases/{id}/similar (similar historical cases), /alerts (polled alerts), and WS /stream (real-time push of stage transitions and new cases — falls back to 30s polling on disconnect).

The detection engine runs a three-layer architecture: Layer 1 Anomaly Detectors (per-layer z-score + data_quality) → Layer 2 Pattern Classifier (M1-M8 rule engine, ML in future iterations) → Layer 3 Lifecycle Tracker (FSM: suspected → accumulate → markup → distribute → collapse → completed / false_alarm). The case repository is in-memory in v1 — active cases are lost on restart; historical cases are rediscovered offline by the HistoricalManipulationScanner.
