Structure Matrix · HTF Tribunal

Page · Structure · 5 min

The central battleground of multi-timeframe structure defense — a 4x3 evidence matrix that shows every symbol's structural integrity across timeframes and zone types, governed by the 8-state MTF Guard state machine.

TL;DR

The Structure Matrix (also called the HTF Tribunal) visualizes multi-timeframe structural analysis as a grid. Rows are timeframes (5m / 15m / 1H / 4H), columns are zone types (OB / FVG / LP). Each cell shows strength and status for that timeframe-zone pair. Above the matrix is a countdown ring showing remaining time on the slowest candle. The verdict panel to the right issues one of five rulings: Allow, Observe, Require Confirmation, Reduce Size, or Block Entry. Below the matrix sit the Shadow Windows panel, charges list, and hearings timeline.

Who uses it & when

Quantitative traders managing multiple strategies who need multi-timeframe confirmation before execution. Open it after a signal fires but before placing the order — the MTF Guard reviews the symbol's structural integrity across timeframes to ensure the trade direction aligns with higher-timeframe structure. Also useful during live positions to periodically check whether the structure is still valid.

Page Anatomy

  1. ① Fast Track Health — A status bar showing data latency, data age, and Redis connectivity. Green means the verdict is trustworthy; orange means data may be stale.
  2. ② Evidence Matrix — 4 rows (timeframes) by 3 columns (zone types). Each cell shows the strength percentage and status. Green = healthy, yellow = caution, red = broken. Cells with diagonal orange stripes are in a shadow window (fast-TF price invaded a slow-TF zone).
  3. ③ Countdown Ring — Shows remaining time until the slowest candle (e.g. 1H) closes. The ring fills as the candle progresses. When the candle closes, the MTF Guard issues its final verdict.
  4. ④ Verdict Panel — The MTF Guard's ruling: Allow, Observe, Require Confirmation, Reduce Size, or Block Entry. A button lets you apply the verdict directly to the order form.
  5. ⑤ Shadow Windows — Active observation windows triggered when fast-TF price enters a slow-TF zone. Each card shows the fast-to-slow pair, violation count, reclaim count, and a candle progress bar.

Reading the Key Metrics

  • Cell Colors — Green (strength >= 70%) means the structure is solid; yellow (40-70%) means caution; red (< 40%) means the structure may be broken. If a whole column (e.g. OB) is green from 4H down to 5m, the zone is respected across all timeframes — a highly reliable signal.
  • Shadow Window Marker — Orange diagonal stripes with an "IN SHADOW" label indicate the fast-TF price has invaded a slow-TF structure zone. This is a temporary state — the verdict is pending until the slow candle closes.
  • Verdict Verbs — Allow = trade freely; Observe = wait, do not act yet; Require Confirmation = need additional confirmation before entry; Reduce Size = trade but with reduced position size; Block Entry = prevent any new positions.
  • Charges List — Problematic cells are summarized as numbered "charges" in Roman numerals. Each charge includes the timeframe, zone type, and a machine-readable reason code (e.g. fast_tf_entered_htf_zone).

Common Actions, Step by Step

  1. Select your symbol and base timeframe at the top (usually the strategy's primary timeframe).
  2. Check the Fast Track Health bar — do not trust the verdict if data is stale.
  3. Scan the evidence matrix: is your trade direction consistently supported across timeframes? If 4H and 1H are green but 5m is red, it may just be short-term noise.
  4. Read the verdict panel — if it says "Block Entry", do not force a trade.
  5. If shadow windows are active, watch the countdown ring. Return when it expires to see if the verdict changes.
  6. Tap any cell to open the detail drawer with full metrics: strength, fill ratio, status, and reason codes.

Behind the Scenes

Data loads through five parallel requests: /api/structure/matrix (grid data), /api/structure/shadow-windows, /api/structure/mtf-guard (guard state and verdict), /api/structure/mtf-guard/events (history), and /api/structure/fast-track-health. The ViewModel uses async let for concurrent fetching. Follows the Redis → service computation → mock data fallback pattern. The backend MTF Guard state machine (MTFTemporalGuardService) manages 8 states (Inactive, Watching, Temporary Violation, Reclaim Pending, Pending HTF Close, Confirmed, Invalidated, Expired) and 6 actions (Allow, Observe, Require Confirm, Block Entry, Reduce Size, Ignore). The Shadow Window service (ShadowWindowService) tracks fast-TF candle progression within each observation window.
