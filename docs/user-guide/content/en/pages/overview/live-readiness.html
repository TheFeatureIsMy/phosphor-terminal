Live Readiness Check

Page · Overview · 5 min

Before going live, the system runs a comprehensive check on infrastructure, exchange connectivity, and performance metrics — and tells you whether you are cleared to trade.

TL;DR

The Live Readiness Check is a safety gate. It aggregates the health of all critical systems (Fast Track, Redis, Freqtrade, exchange API), computes a 0-100 readiness score, and tells you exactly which trading modes are allowed: paper only, small-size live, or full live. Any blockers prevent you from advancing to higher-risk modes.

Who uses it & when

Every time you are about to switch from paper trading to live, or whenever you suspect a system issue. It is also a good practice to run a readiness check before deploying a new strategy to confirm exchange connectivity and that network latencies are within thresholds. During daily operations, if the Dashboard status bar shows an anomaly, come here to see exactly what is wrong.

Page Anatomy

  1. ① Readiness Score Ring — A circular progress bar showing the composite score (0-100). Green at ≥80, yellow at 50-79, red below 50. The score is a weighted calculation from all individual check states.
  2. ② State Banner — Shows the current readiness state (LIVE_READY / LIVE_SMALL_READY / PAPER_ONLY / NOT_READY). Below it, three check badges indicate availability for Paper, Live Small, and Full Live modes — green checkmark means allowed, red X means blocked.
  3. ③ Blockers and Warnings — "Blockers" on a red background must be resolved to advance the readiness level. "Warnings" on a yellow background are advisory. Each item shows an error code and a descriptive message.
  4. ④ System Check Grid — A 4-column grid displaying each subsystem's health: Fast Track latency, Redis round-trip time, Freqtrade status, and Exchange API weight. Each cell contains a status dot (green/yellow/red), metric name, current value, and threshold.
  5. ⑤ Action Bar — "Re-Check" button triggers a fresh health check. "Start Paper Trading" and "Start Small-Size Live" buttons are enabled when the corresponding mode is allowed.

Reading the Key Metrics

  • Readiness Score (0-100) — The most important global metric. Above 80 unlocks full live trading, 50-79 allows small-size live only, below 50 restricts to paper trading. The score is most heavily influenced by the worst check — a single "failed" check drags the entire score down significantly.
  • Fast Track Latency — Internal communication round-trip time. Ideal: <200ms. High latency suggests infrastructure bottlenecks.
  • Redis RTT — Redis cache response time. Ideal: <50ms. Slow Redis affects all cache-dependent data reads.
  • Freqtrade Status — The core trading engine must be "running" for any live mode. Any other status ("stopped", "error") is a blocker.
  • Exchange API Weight — Remaining API quota percentage. Below 90% triggers a warning; dropping below the threshold may cause order submissions to be rate-limited.
  • Readiness State — Four states: LIVE_READY (green, full trading), LIVE_SMALL_READY (orange, small-size only), PAPER_ONLY (yellow, simulation only), NOT_READY (red, untradeable).

Common Actions, Step by Step

Pre-launch check

  1. Open the Live Readiness Check page. Look at the score ring in the top-left — is it in the green zone (≥80)?
  2. Check the "Blockers" section for red items. If any exist, note the error codes and resolve them one by one.
  3. Review the "Warnings" section and decide whether they are acceptable. For example, "Exchange API weight low" is a warning but not a blocker — though it deserves attention in high-frequency scenarios.
  4. Confirm the check badges for your target mode are green (e.g., Full Live shows a checkmark).
  5. Click "Re-Check" to trigger a fresh health check and confirm recently fixed issues are resolved.
  6. Click the action button for the allowed mode (e.g., "Start Small-Size Live") to begin trading.

Behind the Scenes

The page fetches data from the /api/overview/live-readiness endpoint via the APIOverview service. The backend computes the score and state by combining Fast Track heartbeat, Redis connectivity test, Freqtrade process status, and exchange API remaining weight. Clicking "Re-Check" calls the same endpoint to force a refresh. When the backend is unreachable, mock data is returned with a default score of 86 and state LIVE_SMALL_READY.
