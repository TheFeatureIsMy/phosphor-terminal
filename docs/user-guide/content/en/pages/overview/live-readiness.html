Live Readiness Cockpit

Page · Overview · 5 min

Single-screen judgment: 5-level grand status + 11 gates + triple launch confirmation. Any missing infrastructure / strategy / capital / risk / exchange blocks live entry.

TL;DR

Live Readiness is a single-screen judgment control panel. It aggregates 11 critical gates (mode / strategy / capital / risk_config / exchange / data_source / validation / backtest / dryrun / notification / emergency_stop) and derives a 5-level grand status: not_live → needs_config → needs_validation → paper_passed → ready_for_live. Live launch must pass triple confirmation (launch summary → acknowledge → type I confirm live trading) to prevent accidental triggers.

Who uses it & when

Every time you switch from paper trading to live. After deploying a new strategy / changing the capital pool / switching exchange. During daily operations, if the top-bar status strip shows an anomaly, come here to see exactly which gate failed.

Page Anatomy (Single-Screen Layout)

  1. ① Header + 5-level grand status badge — color reflects the state: green (ready_for_live) / cyan (paper_passed) / amber (needs_validation) / yellow (needs_config) / red (not_live). Click "Re-Check" to force a fresh POST /api/overview/live-readiness/check.
  2. ② Selection — 4 pickers — mode / strategy / capital pool / exchange. Switching auto-triggers a re-check to recompute grand_status.
  3. ③ Gates — 11 named checks — 2 columns grouped by category. Left: mode / strategy / capital / risk. Right: system / execution. Each chip shows "dot + label + value"; green = OK, amber = WARN, red = FAIL.
  4. ④ Context — system context — 6 chip grid (notifications / AI models / data source / exchange / freqtrade / redis) + 2 risk gauges (daily used / weekly used) + emergency stop chip. All from real endpoints.
  5. ⑤ Launch — launch authorization — blocker list (if any) + 3 launch buttons (Start Paper / Start Live Small / Start Full Live). Buttons enable/disable based on canStartPaper / canStartLiveSmall / canStartFullLive.

5-Level Grand Status

  • not_live — infrastructure / DB / Redis / data source is unavailable. Fix before continuing.
  • needs_config — any of mode / strategy / capital / risk / exchange is unset.
  • needs_validation — DSL validation / backtest / dry-run has not passed.
  • paper_passed — dry-run is healthy AND mode=paper.
  • ready_for_live — all gates passed AND mode=live_small/full.

Triple Launch Confirmation

  1. Step 1: Review launch summary — sheet shows mode / strategy / exchange / capital / grand_status.
  2. Step 2: Acknowledge — must check "I understand" to advance.
  3. Step 3: Type the phrase — must type I confirm live trading to enable LAUNCH.
  4. Real call to APIEmergency.emergencyStop + reload loadData().

Reading the Key Metrics

  • 5-level grand status — at-a-glance decision: can I go live? Color + badge + description all reinforce.
  • 11 gates — each shows OK / WARN / FAIL + current value + threshold. Any FAIL is a blocker.
  • Context — 6 chips let you confirm platform health (notifications / AI models / data source / exchange / freqtrade / redis).
  • Risk gauges — daily / weekly loss usage. Red = approaching the breaker.

Common Actions, Step by Step

Pre-launch check

  1. Open the Live Readiness Cockpit. Look at the top grand status badge color (green = ready for live).
  2. Confirm all 4 pickers have a selected value (mode / strategy / capital pool / exchange).
  3. Scan the 11 gates: any FAIL = blocker, identify and fix.
  4. Inspect the 6 context chips: all green = platform healthy.
  5. Click "Re-Check" to force a fresh health check (POST /api/overview/live-readiness/check).
  6. Click the appropriate launch button ("Start Live Small", etc.) to open the triple confirmation sheet.

Behind the Scenes

The page fetches data via GET /api/overview/live-readiness?mode=...&strategy_id=... using the APIOverview service. The ViewModel runs 7 sources in parallel (readiness / notifications / overview / ai_models / risk / capital). The backend combines 11 _check_* methods + _derive_grand_status() to derive the 5-level grand status.
