Dashboard Cockpit

The Dashboard is AlphaLoop's production cockpit. Every metric comes from a real backend API: account, strategies, positions, risk, decisions, alerts, provider health, AI models, recent signals. The top bar shows current mode and platform-wide status. No placeholder values, no fabricated numbers.

Top Bar

The top bar is split into two rows, persistent across the app:

Row 1 — Brand + Mode + Primary Actions

  • Brand on the left: "弈机 AlphaLoop" (shown only on the Dashboard route; other routes show a breadcrumb instead)
  • Mode Pill shows LIVE / PAPER / DRYRUN / STOPPED / MOCK / NOT READY in real time, with a color that changes by state
  • Primary actions: language toggle / theme toggle / command palette (⌘K) / notifications (unread badge) / emergency stop (calls APIEmergency directly)

Row 2 — Status Strip (DashboardStatusStrip)

7 status chips in a single row:

  • PROVIDERS — shows healthy/total ratio
  • EXCHANGE — OK / WARNING / DOWN
  • REDIS — RTT in milliseconds
  • FREQTRADE — HEALTHY / WARNING / DOWN
  • RISK — NORMAL / WARNING / LOCKED
  • POSITIONS — current open position count
  • LAST UPDATE — last refresh time (15s polling)

Page Header

Fixed at the top of the Dashboard route:

  • // 驾驶舱 COCKPIT — single page title (avoids duplication with the top bar)
  • Data source badge: LIVE (green) / DATA SOURCE UNAVAILABLE (amber)
  • Subtitle: real-time cockpit — every metric is sourced from the live backend

Card Layout (Bento Grid)

DashboardStatusBar — Infrastructure Status

Single row, 4 cells: System / Freqtrade latency / Redis latency / Exchange. Backed by the BFF system field.

AccountHeroCard — Account Overview (Hero)

Left: large equity counter (with numeric text animation) + 24h change + ModePill + sparkline (from /api/dashboard/equity-curve).

Right: 5 KPI columns — today P&L / week P&L / total P&L (from KPIs) / win rate (from KPIs) / Sharpe ratio. Equity uses BFF account.equity; when both sources are zero, shows — + a DATA SOURCE UNAVAILABLE badge.

AvailableActionsRow — Suggested Actions

Up to 3 context-aware action buttons, sourced from the BFF availableActions. Clicking dispatches to viewModel.performAction(_:): emergency stop calls APIEmergency.emergencyStop directly; start-live / cancel-all navigate to the corresponding page.

Three-Column Metric Cards (Runtime / Readiness / Risk)

  • StrategyRuntimeCard — running strategy count + 3 colored dots for positions / pending / reconciling
  • LiveReadinessCard — readiness lamp (Live Ready / Paper Only / Risk Locked / Emergency Locked) + the real checks list (no more 7 fabricated OK chips)
  • GlobalRiskCard — global risk state pill + daily / weekly loss-remaining gauges

PositionRiskTable — Position Risk

Full-width table. Sourced from GET /api/execution/positions (mockPositions hardcoded array removed). Columns: symbol / direction (left stripe) / size / entry / mark / P&L / P&L % / state badge (in_sync / drift / local_only / exchange_only). Empty state when no positions; DATA SOURCE UNAVAILABLE when backend is down.

ProviderHealthCard + AIModelStatusCard

Two-column grid:

  • Provider: healthy / warning / error counts + per-entry expansion (name, latency, last error)
  • AI models: each model's provider / state / GPU memory

SignalsFeedCard — Signal Feed (with Source)

Sourced from GET /api/agent-signals/signals. Every signal explicitly shows 3 source-trace tags: AGENT / STRATEGY / SNAPSHOT (or NO TRACE when missing).

RecentDecisionFeed + AlertTimeline

Two-column layout. BFF recent_decisions and alerts; empty arrays render as 暂无决策 / 暂无告警 respectively.

EmergencyActionBar — Emergency Control

Pinned at the bottom: icon + description + red "Stop All Trading" button. Click → confirmation dialog → real APIEmergency.emergencyStop.

LearnAlphaLoopCard — Learn Card

Shown only when state == "unknown" and there is no data. Opens the user guide. Can be dismissed.

Data Refresh

30s page-level polling. 15s top-bar strip polling. Emergency button click → immediate APIEmergency.emergencyStop → toast feedback → reload load().

Data-Source Failure Fallback

If any sub-endpoint fails:

  • The affected card renders an empty state with the failure tag; other cards are unaffected
  • If the overall BFF returns state == "data_source_unavailable", the page renders DataSourceUnavailableView (icon + title + retry button)
