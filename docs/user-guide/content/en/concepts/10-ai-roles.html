AI Roles in AlphaLoop

Concept · 10 / 10 · 6 min

AI in AlphaLoop isn't one all-knowing model — it's four specialized roles, each doing one thing, chained into a pipeline.

Why It Matters

A common misconception about AI in quant: one big model can "give me a strategy" or "tell me what to buy." Reality: large models aren't good enough at any single step. Ask GPT to write strategy code directly and it produces something that runs but is wildly overfit; ask it to analyze markets and it hallucinates about real-time data it can't see.

AlphaLoop's design principle: split AI into 4 small roles, each doing only what it's best at, leaving the rest (precise math, real-time data, rigorous backtesting) to traditional tools. This harnesses the inductive power of LLMs without letting them do anything unreliable.

The Four Roles

  • ① Research — Reads papers, reports, blogs, and distills them into concrete, tradeable rules. E.g. reading an SMC paper produces "after a 4H BOS, wait for pullback into the 1H Order Block, then long." Powered by the TradingAgents multi-agent framework.
  • ② Generation — Translates that description into a Freqtrade-runnable Python strategy and runs a quick backtest for sanity. This step also runs static code safety checks (no eval / no network / no file writes) to prevent generated code from doing anything unexpected.
  • ③ Fusion — At any moment multiple strategies / signals will say "buy BTC" while others disagree. The fusion layer scores each signal (by that strategy's last 30 days) and produces a weighted consensus. High score + directional agreement = top priority.
  • ④ Sentiment — Uses FinBERT (a BERT model trained on financial text) to score real-time news and Twitter, giving each asset a sentiment score from -1 to +1. Sentiment aligned with structural signals increases size; misalignment requires a stronger entry case.

Common Pitfalls

  • Treating AI as an oracle — it isn't. Each role makes errors; the pipeline design ensures single-point failures don't directly lose money (signals have score thresholds, code has safety checks, sentiment is a modifier only).
  • Skipping human review — AI-generated strategies must be eyeballed (code logic + backtest curve) before going live. Don't blind-trust.
  • Trading sentiment contrarianly — FinBERT isn't a crystal ball; by the time sentiment is extreme, markets often already reflect it. Use sentiment to strengthen or weaken existing signals — never as a standalone trigger.
  • Assuming AI replaces thinking — AI is an amplifier: right judgment, amplified results; wrong judgment, amplified losses, faster.

In AlphaLoop

  • AI Research Room — upload PDFs / reports, let the research agent extract strategies
  • Agent Platform — configure and monitor multiple AI agents
  • Signal Center — view fused signals and their raw inputs
  • Market Sentiment — FinBERT sentiment heatmap
