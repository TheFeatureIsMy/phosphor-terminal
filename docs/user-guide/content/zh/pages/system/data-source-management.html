数据源管理

页面 · 系统 · 5 分钟

管理所有接入 AlphaLoop 的数据管道——行情 K 线、订单簿、资金费率、链上数据、新闻与社交情绪——集中监控延迟、新鲜度，支持启用/禁用和连接测试。

TL;DR

数据源管理页面是 AlphaLoop 所有数据管道的控制中心。它按类别（行情 K 线、订单簿、资金费率、持仓量、新闻、巨鲸、链上数据、研究报告、社交情绪）对数据源分组，每张卡片展示延迟、新鲜度、最后获取时间和状态。顶部汇总卡片显示总数、活跃数、异常数和分类数。你可以测试单个连接、批量测试所有活跃数据源，或一键切换启用/禁用。

谁用 · 何时用

配置新交易所或数据提供商后，来这里确认连接正常。当策略信号基于某个数据源但结果异常时，检查该数据源是否过期或状态异常。也适合在系统迁移后批量测试所有数据源连通性。

页面解剖

  1. ① 汇总卡片 — 总数据源数、活跃数、异常数、分类数。异常数 > 0 时应优先排查。
  2. ② 分类筛选 — 按类别过滤数据源。支持行情 K 线、订单簿、资金费率、持仓量、新闻、巨鲸、链上数据、研究报告、社交情绪。
  3. ③ 数据源卡片 — 每张卡片展示：名称、状态指示、提供商、延迟（ms）、新鲜度（新鲜/陈旧/过期）、最后获取时间。底部有"测试"和"启用/禁用"按钮。
  4. ④ 状态警告横幅 — 当有数据源状态异常时自动显示，列出异常原因码以便快速定位。
  5. ⑤ 批量测试 — 页面标题旁的"测试所有连接"按钮，逐一对所有活跃数据源执行连通性测试。

关键指标怎么读

  • 延迟 (Latency) — 从发送请求到收到响应的耗时。绿色 < 100ms，黄色 100-300ms，橙色 > 300ms。持续高延迟可能需要检查网络或提供商状态。
  • 新鲜度 (Freshness) — 最近一次数据获取距今的时间。绿色 = 新鲜（数据及时），黄色 = 陈旧（略有延迟），红色 = 过期（数据不可信）。过期数据会导致策略信号失真。
  • 状态 (Status) — 活跃（正常采集）、已禁用（手动关闭）、异常（连接失败）、限流（触发频率限制）。异常和限流需要人工介入。
  • 最后获取时间 — 相对时间显示（刚刚 / 5 分钟前 / 2 小时前），帮助你判断数据是否仍在流动。

常见操作 · 一步步

  1. 新增数据源后，在对应分类下找到新卡片，点击"测试"验证连接成功。
  2. 如测试失败，检查 API Key 是否有效、网络是否可达、提供商是否限流。
  3. 暂时不需要的数据源可点击"禁用"关闭，避免无效请求浪费资源。
  4. 批量操作：点击"测试所有连接"逐一验证所有活跃数据源。
  5. 查看异常数据源的原因码（如 rate_limit_exceeded），按提示处理。

幕后

页面数据通过 ViewModel 的 load() 方法获取，调用后端 API 返回所有数据源的状态和指标。每次"测试"操作触发独立的连通性检测请求，结果实时更新在卡片上。"测试所有连接"按顺序逐一对活跃数据源执行测试。启用/禁用切换调用后端更新数据源配置。分类筛选是本地过滤，不触发新网络请求。
