设置

页面 · 系统 · 5 分钟

AlphaLoop 所有偏好的集中配置中心——语言、个人资料、交易所、风控、通知、API 密钥、MCP 服务、数据清理和高级操作。

TL;DR

设置页面使用顶部 Tab 栏组织为 7 个分区：通用（语言切换 + 个人资料）、交易（交易所配置 + 风控参数）、通知（Telegram 集成 + 事件开关）、API（第三方 API 密钥）、服务（MCP 服务器状态与审计日志）、数据（数据清理/归档）、高级（数据导出与账户删除）。每个 Tab 切换带动画滑动指示器。所有设置支持本地修改和云端持久化。

谁用 · 何时用

每个用户都会用到。首次启动时配置语言、交易所和 API 密钥。风控参数在策略上线前设定。通知配置在需要 Telegram 告警时设置。数据清理在存储接近上限时执行。高级操作仅管理员使用。

页面解剖

  1. ① Tab 栏 — 7 个分区 Tab：通用、交易、通知、API、服务、数据、高级。选中 Tab 有绿色滑动指示器和发光阴影。
  2. ② 通用设置 — 语言切换（简体中文 / English，即时生效）、个人资料（用户名、邮箱、角色、时区、2FA 状态）。
  3. ③ 交易所配置 — 选择交易所（Binance/OKX/Bybit 等）、交易模式（现货/合约）、模拟交易开关、API Key/Secret 输入。
  4. ④ 自动保存 — 设置修改后 2 秒防抖自动保存到后端。离线时使用本地默认值，不影响操作。
  5. ⑤ 高级操作 — 数据导出（策略和交易记录）和账户删除。删除操作需弹窗二次确认。

关键设置详解

  • 语言切换 — 在简体中文和 English 之间切换。更改后所有页面即时响应新语言，无需重启。
  • 模拟交易 (Dry Run) — 开启后交易所 API 仅用于获取行情，订单不会真实发送。适合策略验证期。
  • 风控参数 — 单笔最大亏损（默认 5%）、最大回撤（15%）、日回撤上限（8%）、自动暂停。超过阈值时系统自动停止开仓。
  • 通知配置 — Telegram Bot Token 和 Chat ID 用于发送交易执行、风险事件、每日摘要和系统告警通知。
  • MCP 服务 — 查看 MCP 服务器状态（绑定地址、请求计数）、审计日志（每次工具调用的响应状态码和延迟），支持 Token 轮换。
  • 数据清理 (Vacuum) — 扫描并归档过期信号数据，释放存储空间。可查看清理历史记录（已扫描数和已归档数）。

常见操作 · 一步步

  1. 首次使用：切到"通用"Tab 选择语言，填写个人资料。切到"交易"Tab 配置交易所和 API 密钥，确认模拟交易已开启。
  2. 配置风控：在"交易"Tab 设定单笔最大亏损和最大回撤，打开自动暂停开关。
  3. 连接 Telegram：在"通知"Tab 输入 Bot Token 和 Chat ID，勾选你感兴趣的事件类型。
  4. 配置 AI 服务：在"API"Tab 填入 OpenAI 或 Anthropic 的 API Key。在"服务"Tab 查看 MCP 审计日志。
  5. 数据清理：当存储空间不足时，在"数据"Tab 点击"执行清理"。可在清理历史中查看执行结果。
  6. 恢复默认风控参数：在"交易"Tab 底部点击"恢复默认"。所有风控值回到出厂设置。

幕后

设置状态由 SettingsState 管理（单例，通过依赖注入初始化）。首次启动时调用 /auth/settings 从后端拉取已保存的设置并覆盖本地默认值。每次修改触发 scheduleSave()——2 秒防抖窗口内无新变更时推送 /auth/settings 更新。语言切换通过 L10n 系统全局生效，视图通过 .id(settings.language) 强制刷新。离线时所有设置保留本地默认值，不阻塞用户操作。
