回测与模拟

页面 · 策略 · 5 分钟

在投入真金白银之前，用历史数据检验你的策略表现，或在模拟环境中观察它的实时行为。

TL;DR

回测与模拟是验证策略的两个阶段。BacktestLab 页面采用双栏布局：左侧 Run Rail（240pt）列出历史回测记录，右侧是一个可滚动的叙事流面板，包含 9 个依次解锁的区域。回测经历 idle → configuring → running → completed → failed 五个阶段，区域 3-8（Summary、Curve、TradeList、Compare、Risk、Promotion）仅在 completed 后可见。页面支持对比模式（Run Rail 中勾选最多 3 个回测），风险面板检查 5 项规则（最大回撤、盈亏比、交易数、胜率、夏普比率），失败聚类分为运行级和策略级两个层次。底部的推广门汇总 11 项就绪检查，CTA 跳转到 Live Readiness 页面（不从此页面启动实盘交易）。

谁用 · 何时用

所有策略作者。回测是上线前的必由之路 — 每次修改 DSL 规则或画布后，都应该运行一次回测。BacktestLab 页面（从侧边栏进入）提供完整的回测引擎界面，包含 Run Rail 历史记录和叙事流面板。模拟（纸交易）则在回测结果满意后、上线前的最后一步使用。

页面解剖

  1. ① Run Rail — 左侧 240pt 的竖栏，列出该策略的历史回测记录。每条记录显示序号、日期和收益率（绿色=盈利，红色=亏损）。每条记录左侧有复选框，最多勾选 3 条进入对比模式。列表顶部的"对比模式 (N/3)"徽章显示当前选中数。
  2. ② 叙事流面板 — 右侧滚动区域，按顺序排列 9 个区域。区域 1（ConfigPanel）和区域 2（StatusPanel）在回测运行期间可见；区域 3-8 仅在回测完成后解锁（completed 阶段）；区域 9（DataSourceFooter）始终可见。

区域参考

  1. ConfigPanel — 回测配置区。设置时间范围（YYYY-MM-DD 格式）、初始资金（USDT）、交易对（逗号分隔）。策略版本自动从当前策略获取。点击"启动回测"提交任务。始终可见。
  2. StatusPanel — 执行进度面板。回测运行中显示阶段指示器（configuring→running）和已用时间。完成后自动切换为结果展示。
  3. SummaryPanel — 8 项核心指标卡：总收益率、夏普比率、最大回撤、胜率、盈亏比、总交易数、初始资金、交易对。每个指标用颜色区分好坏。仅 completed 阶段可见。
  4. CurvePanel — 权益曲线图。显示回测期间资金变化趋势。仅 completed 阶段可见。
  5. TradeListPanel — 逐笔交易明细表。列出每笔交易的时间、方向、价格、数量和盈亏。仅 completed 阶段可见。
  6. ComparePanel — 对比面板。在 Run Rail 中勾选 2-3 条记录后展开，显示对比矩阵（各指标并排比较）和权益曲线叠加图。仅 completed + 勾选 ≥2 条时可见。
  7. RiskPanel — 风险规则面板。检查 5 项规则：最大回撤阈值、盈亏比下限、最低交易数、胜率下限、夏普比率下限。每项规则显示 pass / warn / fail。仅 completed 阶段可见。
  8. PromotionPanel — 推广门面板。汇总 11 项就绪检查（策略版本、回测通过、风险合规等），显示通过/未通过状态。CTA 按钮跳转到 Live Readiness 页面 — 不从此页面启动实盘交易，仅作为就绪度入口。
  9. DataSourceFooter — 数据源信息栏。显示 Freqtrade 引擎版本、OHLCV 数据源、回测执行耗时、DSL 哈希值。始终可见。

关键指标怎么读

  • 总收益率 (Total Return) — 回测期间的总盈利百分比。正值为绿色，负值为红色。结合最大回撤一起看：40% 的收益配 5% 的回撤比 40% 配 25% 好得多。
  • 夏普比率 (Sharpe Ratio) — 衡量风险调整后收益。≥1.5 优秀（绿色），1.0-1.5 可接受（黄色），<1.0 偏低。它告诉你在承受每单位波动时获得了多少超额收益。
  • 最大回撤 (Max Drawdown) — 回测期间从峰值到谷底的最大跌幅。总是显示为红色。加密货币策略通常回撤 20-40% 可接受，但取决于策略类型。
  • 胜率 (Win Rate) — 盈利交易占总交易的比例。≥50% 为绿色。但胜率并非越高越好 — 高胜率搭配低盈亏比可能还不如低胜率高盈亏比。
  • 盈亏比 (Profit Factor) — 总盈利 / 总亏损。≥1.5 为绿色。如果盈亏比高，即使胜率只有 40%，策略仍然可能盈利。

常见操作 · 一步步

  1. 运行回测 — 从侧边栏进入 BacktestLab 页面。确认策略已有保存的版本。在 ConfigPanel 中填写时间范围、初始资金、交易对。点击"启动回测"。
  2. 查看进度 — 回测异步执行。StatusPanel 显示当前阶段和已用时间。前端每 3 秒轮询一次状态。
  3. 解读结果 — 回测完成后 SummaryPanel（指标卡）、CurvePanel（权益曲线）、TradeListPanel（交易明细）自动出现。查看夏普比率和最大回撤判断策略是否值得优化。
  4. 对比多个回测 — 在 Run Rail 中勾选 2-3 条历史记录，ComparePanel 自动展开。对比矩阵并排显示各指标，权益叠加图展示多条曲线的差异。
  5. 检查风险 — RiskPanel 对 5 项规则进行 pass/warn/fail 评估。红色标记的规则需要优化后重新回测。
  6. 检查就绪度 — PromotionPanel 汇总 11 项就绪门。点击 CTA 跳转到 Live Readiness 页面查看完整就绪度详情（不在此页面启动实盘交易）。
  7. 查看数据来源 — DataSourceFooter 显示回测使用的引擎、数据源和执行信息，便于审计和复现。

失败聚类

回测失败分两个层次排查：运行级失败 — 数据源不可用、引擎崩溃、超时，此类失败重启即可；策略级失败 — DSL 语法错误、策略逻辑异常、指标计算错误，此类失败需要修改策略后重新提交。StatusPanel 会明确标注失败层级，帮助快速定位问题。

模拟运行（纸交易）

纸交易监控在独立页面中运行。策略在模拟环境中接收实时市场数据并产生信号，但不执行真实订单。从侧边栏进入 Dryrun Monitor 页面查看活跃的模拟运行，每个运行卡片显示模式、状态、开始时间和最新信号。

幕后

回测通过 Command Bus 异步执行。点击"启动回测"后，前端调用 api.startBacktest() 提交任务，后端返回 commandId。前端随后每 3 秒轮询 api.backtestStatus(commandId:)。页面状态机经历 idle → configuring → running → completed / failed 五个阶段，区域 3-8 根据状态机阶段解锁显示。对比模式下，前端收集选中回测的 runId 列表，调用 api.compareBacktests(runIds:) 获取对比矩阵数据。风险规则在前端本地计算，基于回测结果中的 KPI 与预设阈值比较。模拟运行通过 APIStrategyRuns.listRuns(mode: "dryrun") 获取。
