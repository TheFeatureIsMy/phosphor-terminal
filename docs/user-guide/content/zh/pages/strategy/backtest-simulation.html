回测与模拟

页面 · 策略 · 5 分钟

在投入真金白银之前，用历史数据检验你的策略表现，或在模拟环境中观察它的实时行为。

TL;DR

回测与模拟是验证策略的两个阶段。回测使用历史数据（可指定时间范围、初始资金、交易对）运行策略，计算总收益率、夏普比率、最大回撤、胜率等指标，并展示权益曲线和逐笔交易明细。模拟（纸交易）让策略在实时市场中运行但不下真实订单，你可以观察它的表现和信号。两者都在策略详情页的"回测"和"模拟"标签页中操作。回测支持 Command Bus 轮询模式 — 提交后系统异步执行，前端每 3 秒轮询一次状态。

谁用 · 何时用

所有策略作者。回测是上线前的必由之路 — 每次修改 DSL 规则或画布后，都应该运行一次回测。模拟则是在回测结果满意后、上线前的最后一步：让策略在模拟环境中运行几天到几周，观察实际市场下的表现。独立页面（从侧边栏进入）提供更完整的回测引擎界面，包含历史回测记录列表。

页面解剖

  1. ① 回测配置区 — 设置时间范围（YYYYMMDD-YYYYMMDD 格式）、初始资金（USDT）、交易标的（逗号分隔）。策略版本从当前策略自动获取。点击"启动回测"提交任务。
  2. ② 回测结果指标卡 — 运行完成后显示 8 项核心指标：总收益率（绿/红）、夏普比率（≥1.5 绿）、最大回撤（红色标注）、胜率（≥50% 绿）、盈亏比（≥1.5 绿）、总交易数、初始资金、交易对。每个指标用颜色区分好坏。
  3. ③ 权益曲线与交易明细 — 简化的条形图显示资金变化趋势（绿色=盈利日，红色=亏损日）。下方表格列出最近交易的时间、方向、价格、数量和盈亏。

关键指标怎么读

  • 总收益率 (Total Return) — 回测期间的总盈利百分比。正值为绿色，负值为红色。结合最大回撤一起看：40% 的收益配 5% 的回撤比 40% 配 25% 好得多。
  • 夏普比率 (Sharpe Ratio) — 衡量风险调整后收益。≥1.5 优秀（绿色），1.0-1.5 可接受（黄色），<1.0 偏低。它告诉你在承受每单位波动时获得了多少超额收益。
  • 最大回撤 (Max Drawdown) — 回测期间从峰值到谷底的最大跌幅。总是显示为红色。加密货币策略通常回撤 20-40% 可接受，但取决于策略类型。
  • 胜率 (Win Rate) — 盈利交易占总交易的比例。≥50% 为绿色。但胜率并非越高越好 — 高胜率搭配低盈亏比可能还不如低胜率高盈亏比。
  • 盈亏比 (Profit Factor) — 总盈利 / 总亏损。≥1.5 为绿色。如果盈亏比高，即使胜率只有 40%，策略仍然可能盈利。

常见操作 · 一步步

  1. 运行回测 — 在策略详情页进入"回测"标签页（或从侧边栏进入独立的回测引擎页面）。确认策略已有保存的版本（没有则先在 DSL 规则中保存）。填写时间范围、初始资金、交易对。点击"启动回测"。
  2. 查看结果 — 回测异步执行，前端每 3 秒轮询一次状态。完成后指标卡和权益曲线自动出现。查看 Sharpe 比率和最大回撤来判断策略是否值得进一步优化。
  3. 启动模拟运行 — 在策略详情页进入"模拟"标签页。系统自动列出该策略的活跃模拟运行。每个运行卡片显示模式、状态、开始时间。运行中（绿色）表示策略在模拟环境中实时产生信号。
  4. 查看历史回测 — 在独立的回测引擎页面，底部显示最近 10 次回测记录。每行显示交易对、时间范围、收益率和 Sharpe 比率。点击可查看详细结果。

幕后

回测通过 Command Bus 异步执行。点击"启动回测"后，前端调用 api.startBacktest() 提交任务，后端返回一个 commandId。前端随后每 3 秒调用 api.backtestStatus(commandId:) 轮询状态。当 commandStatus 变为 completed 时，结果被解析为 BacktestRunV2 对象并在 BacktestResultCardView 中显示。模拟运行通过 APIStrategyRuns.listRuns(mode: "dryrun") 获取，过滤出当前策略的运行记录。模拟状态（运行中/已完成/已停止/失败）由后端实时更新。
