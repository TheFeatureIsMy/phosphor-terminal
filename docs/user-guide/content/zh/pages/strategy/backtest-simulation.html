回测与模拟

页面 · 策略 · 5 分钟

在投入真金白银之前，用历史数据检验你的策略表现，或在模拟环境中观察它的实时行为。

TL;DR

BacktestLab 页面采用单列数据终端布局（2026-07-01 重新设计）。48pt 顶 bar 包含 RunSwitcher 切换器 + Backtest/Dryrun 分段切换 + 新建回测按钮 + 对比按钮（勾选 ≥2 条时出现）。下方依次是 360pt EquityCurveHero（权益曲线渐变填充 + 对比叠加）、可折叠的 StrategyContextStrip（策略+风险+晋升门摘要）、2×4 MetricsGrid（8 项指标卡）和 TradeListTable（紧凑可折叠表格，20 行分页）。右侧抽屉面板：NewRunDrawer（配置表单）和 HistoryDrawer（历史记录+对比勾选），通过 DrawerContainer 滑入。旧版三栏布局已被替换。

谁用 · 何时用

所有策略作者。回测是上线前的必由之路 — 每次修改 DSL 规则或画布后，都应该运行一次回测。BacktestLab 页面（从侧边栏进入）提供完整的回测引擎界面。模拟（纸交易）则在回测结果满意后、上线前的最后一步使用。

页面解剖

  1. ① Top Bar（48pt） — 顶栏左侧 RunSwitcher 按钮显示当前选中运行（编号 · 交易对 · 日期），点击打开历史记录抽屉。中间 Backtest / Dryrun 分段切换开关。右侧"新建回测"按钮打开 NewRunDrawer。当勾选 ≥2 条历史记录时出现"对比"按钮，点击展开 EquityCurveHero 的对比叠加模式。
  2. ② EquityCurveHero（360pt） — 主视觉权益曲线。单运行模式显示渐变填充曲线图，下方三条标签（总收益 / 最大回撤 / 峰值）。对比模式叠加多条曲线，每条使用不同颜色，右上角显示图例。数据为空时显示空白区域。
  3. ③ StrategyContextStrip — 可折叠的策略上下文栏。折叠态显示"策略 #ID · N 条警告 · 晋升门状态"一行摘要。展开后展示策略详细信息、风险警告列表和晋升就绪状态。
  4. ④ MetricsGrid（2×4） — 2 行 4 列共 8 项指标卡：总收益率、最大回撤、夏普比率、胜率、盈亏比、交易次数、实际成交、平均持仓。使用等宽数字字体，正值绿色、负值红色。
  5. ⑤ TradeListTable — 紧凑可折叠交易明细表。默认显示前 20 行，点击"显示全部"展开全部。表头列：编号、时间、方向、入场、出场、盈亏、时长。无成交时显示"无成交记录"。
  6. ⑥ 抽屉面板 — 从右侧滑入的面板，通过 DrawerContainer 管理。NewRunDrawer 包含新回测配置表单（交易对、时间周期、策略 ID）。HistoryDrawer 显示历史运行列表，每条带对比勾选框，支持搜索。

区域参考

  1. RunSwitcher — 顶 bar 左侧的按钮，显示当前选中运行的编号、交易对和日期。点击打开 HistoryDrawer 切换运行。"选择运行"为无选中时的占位文字。
  2. EquityCurveHero — 360pt 高的权益曲线主视觉。使用 surfaceHover.opacity(0.35) 背景 + border 的深色终端风格。单运行显示渐变填充，对比模式叠加多色折线。下方三条内联指标（总收益、最大回撤、峰值）。
  3. StrategyContextStrip — 策略上下文可折叠区域，折叠态一条摘要，展开后显示策略详情、风险警告和推广就绪门状态。
  4. MetricsGrid — 2×4 网格展示 8 项核心回测指标，使用等宽数字字体，颜色表示正负。
  5. TradeListTable — 成交记录表格，可折叠、20 行分页、表头固定。
  6. NewRunDrawer — 右侧抽屉式配置表单，配置交易对、周期、策略 ID 后启动回测或模拟。
  7. HistoryDrawer — 右侧抽屉式历史记录列表，支持搜索和对比勾选，点击行切换焦点。

关键指标怎么读

  • 总收益率 (Total Return) — 回测期间的总盈利百分比。正值为绿色，负值为红色。结合最大回撤一起看：40% 的收益配 5% 的回撤比 40% 配 25% 好得多。
  • 夏普比率 (Sharpe Ratio) — 衡量风险调整后收益。≥1.5 优秀（绿色），1.0-1.5 可接受（黄色），<1.0 偏低。它告诉你在承受每单位波动时获得了多少超额收益。
  • 最大回撤 (Max Drawdown) — 回测期间从峰值到谷底的最大跌幅。总是显示为红色。加密货币策略通常回撤 20-40% 可接受，但取决于策略类型。
  • 胜率 (Win Rate) — 盈利交易占总交易的比例。≥50% 为绿色。但胜率并非越高越好 — 高胜率搭配低盈亏比可能还不如低胜率高盈亏比。
  • 盈亏比 (Profit Factor) — 总盈利 / 总亏损。≥1.5 为绿色。如果盈亏比高，即使胜率只有 40%，策略仍然可能盈利。

常见操作 · 一步步

  1. 运行回测 — 从侧边栏进入 BacktestLab 页面。点击顶 bar 的"新建回测"按钮打开 NewRunDrawer 抽屉，填写交易对、周期和策略 ID，点击"回测"启动。新运行加载到 HistoryDrawer 列表中。
  2. 查看结果 — 选中一个完成的回测，EquityCurveHero 显示权益曲线，MetricsGrid 显示 8 项指标，TradeListTable 显示交易明细。可折叠 StrategyContextStrip 了解策略上下文。
  3. 对比多个回测 — 在 HistoryDrawer 中勾选 2-3 条记录，顶 bar 出现"对比"按钮。点击进入对比模式，EquityCurveHero 叠加显示多条曲线。
  4. 切换模拟模式 — 点击顶 bar 的"模拟"标签切换到纸交易模式。模拟运行的配置方式与回测相同。
  5. 搜索历史记录 — 在 HistoryDrawer 顶部的搜索框输入交易对名称过滤历史运行记录。

幕后

回测通过 Command Bus 异步执行。提交任务后前端轮询状态。页面状态机经历 idle → configuring → running → completed / failed 五个阶段。对比模式下前端收集选中 runId 列表，请求后端对比数据。所有数据面板使用深色终端风格：surfaceHover.opacity(0.35) 背景 + border 边框，等宽 PulseFonts.tabular 数字，无玻璃质感/花体/斜体装饰。
