回测与模拟

页面 · 策略 · 5 分钟

在投入真金白银之前，用历史数据检验你的策略表现，或在模拟环境中观察它的实时行为。

TL;DR

BacktestLab 页面采用三栏布局：左侧 Run Rail（240pt）列出历史回测记录并支持对比勾选，中间是 Backtest/Dryrun 标签切换 + 可滚动的内容面板（配置 → 状态/摘要 → 权益曲线+回撤 → 交易明细+失败聚类 → 对比），右侧 Context Rail（280pt）始终显示策略元信息、风险预警和推广就绪门。页面不再使用旧版 NewRunSheet 弹窗，所有操作均内联在主布局中。

谁用 · 何时用

所有策略作者。回测是上线前的必由之路 — 每次修改 DSL 规则或画布后，都应该运行一次回测。BacktestLab 页面（从侧边栏进入）提供完整的回测引擎界面。模拟（纸交易）则在回测结果满意后、上线前的最后一步使用。

页面解剖

  1. ① Run Rail（左栏） — 240pt 宽的历史回测列表。每条记录显示序号、日期和收益率（绿色=盈利，红色=亏损），右侧有复选框，最多勾选 3 条进入对比模式。底部"新建回测"按钮启动新回测或模拟运行，不再使用旧版弹窗表单。
  2. ② 中间栏 — 顶部是 Backtest / Dryrun 标签切换。下方为可滚动的内容区：ConfigPanel 始终可见；回测完成后依次显示 StatusSummaryBlock（8 项 KPI）、EquityCurveBlock（权益曲线+回撤）、TradeListBlock（交易明细+失败聚类）；勾选 ≥2 条运行后自动展开 CompareBlock（对比矩阵+权益叠加）。
  3. ③ Context Rail（右栏） — 280pt 宽的上下文信息栏，始终可见。包含策略元信息（名称、版本、数据源）、风险预警（5 项规则检查，completed 后显示）和推广就绪门（11 项检查，CTA 跳转到 Live Readiness 页面）。

区域参考

  1. Run Rail — 左侧竖栏列出当前策略的所有历史回测/模拟运行记录。每条记录显示运行编号、日期、收益率和状态徽章。勾选复选框进入对比模式（最多 3 条），CompareBlock 自动在中间栏展开。点击"新建回测"按钮直接在 ConfigPanel 中配置新运行。
  2. ConfigPanel — 回测配置区，位于中间栏顶部。选择策略版本、交易对（逗号分隔）、时间范围（YYYY-MM-DD）、初始资金（USDT）、手续费和滑点。点击"启动"提交任务。始终可见。
  3. StatusSummaryBlock — 执行状态与摘要面板。回测运行中显示阶段指示器（configuring → running）和已用时间。完成后切换为 8 项核心指标卡：总收益率、夏普比率、最大回撤、胜率、盈亏比、总交易数、初始资金、交易对。每个指标用颜色区分好坏。在 completed 或 failed 阶段可见。
  4. EquityCurveBlock — 权益曲线图与回撤图。显示回测期间资金变化趋势及最大回撤区间。仅 completed 阶段可见。
  5. TradeListBlock — 逐笔交易明细表。列出每笔交易的时间、方向、价格、数量和盈亏。同时展示运行级和策略级失败聚类信息。仅 completed 阶段可见。
  6. CompareBlock — 对比面板。在 Run Rail 中勾选 2-3 条记录后自动展开，显示对比矩阵（各指标并排比较）和权益曲线叠加图。仅 completed + 勾选 ≥2 条时可见。
  7. StrategyMetaPanel（右栏） — 策略元信息面板，始终显示当前策略的名称、DSL 版本、数据源（Freqtrade 引擎、OHLCV 源）、回测执行耗时和 DSL 哈希值。
  8. RiskWarningsPanel（右栏） — 风险规则面板，检查 5 项规则：最大回撤阈值、盈亏比下限、最低交易数、胜率下限、夏普比率下限。每项规则显示 pass / warn / fail。回测完成后可见。
  9. PromotionGatePanel（右栏） — 推广就绪门面板，汇总 11 项就绪检查（策略版本、回测通过、风险合规等），显示通过/未通过状态。CTA 按钮跳转到 Live Readiness 页面 — 不从此页面启动实盘交易，仅作为就绪度入口。

关键指标怎么读

  • 总收益率 (Total Return) — 回测期间的总盈利百分比。正值为绿色，负值为红色。结合最大回撤一起看：40% 的收益配 5% 的回撤比 40% 配 25% 好得多。
  • 夏普比率 (Sharpe Ratio) — 衡量风险调整后收益。≥1.5 优秀（绿色），1.0-1.5 可接受（黄色），<1.0 偏低。它告诉你在承受每单位波动时获得了多少超额收益。
  • 最大回撤 (Max Drawdown) — 回测期间从峰值到谷底的最大跌幅。总是显示为红色。加密货币策略通常回撤 20-40% 可接受，但取决于策略类型。
  • 胜率 (Win Rate) — 盈利交易占总交易的比例。≥50% 为绿色。但胜率并非越高越好 — 高胜率搭配低盈亏比可能还不如低胜率高盈亏比。
  • 盈亏比 (Profit Factor) — 总盈利 / 总亏损。≥1.5 为绿色。如果盈亏比高，即使胜率只有 40%，策略仍然可能盈利。

常见操作 · 一步步

  1. 运行回测 — 从侧边栏进入 BacktestLab 页面。确认策略已有保存的版本。在 ConfigPanel 中填写配置项，点击"启动"。回测在左侧 Run Rail 中新增一条记录，状态为 configuring。
  2. 查看进度 — 回测异步执行。StatusSummaryBlock 显示当前阶段和已用时间。前端每 3 秒轮询一次状态。
  3. 解读结果 — 回测完成后中间栏自动展开 StatusSummaryBlock（指标卡）、EquityCurveBlock（权益曲线）、TradeListBlock（交易明细）。查看夏普比率和最大回撤判断策略是否值得优化。
  4. 对比多个回测 — 在 Run Rail 中勾选 2-3 条历史记录，CompareBlock 自动展开。对比矩阵并排显示各指标，权益叠加图展示多条曲线的差异。
  5. 检查风险 — 右栏 RiskWarningsPanel 对 5 项规则进行 pass/warn/fail 评估。红色标记的规则需要优化后重新回测。
  6. 检查就绪度 — 右栏 PromotionGatePanel 汇总 11 项就绪门。点击 CTA 跳转到 Live Readiness 页面查看完整就绪度详情（不在此页面启动实盘交易）。
  7. 查看数据来源 — 右栏 StrategyMetaPanel 显示回测使用的引擎、数据源和执行信息，便于审计和复现。
  8. 切换模拟模式 — 点击中间栏顶部的"模拟"标签切换到纸交易模式。模拟运行的配置方式与回测相同，Run Rail 中显示模拟状态。

失败聚类

回测失败分两个层次排查：运行级失败 — 数据源不可用、引擎崩溃、超时，此类失败重启即可；策略级失败 — DSL 语法错误、策略逻辑异常、指标计算错误，此类失败需要修改策略后重新提交。StatusSummaryBlock 和 TradeListBlock 会明确标注失败层级，帮助快速定位问题。

模拟运行（纸交易）

纸交易监控集成在 BacktestLab 页面中。点击中间栏顶部的"模拟"标签切换到纸交易模式。策略在模拟环境中接收实时市场数据并产生信号，但不执行真实订单。Run Rail 中显示活跃的模拟运行，每个运行卡片显示模式、状态、开始时间和最新信号。

幕后

回测通过 Command Bus 异步执行。点击"启动"后，前端调用 api.startBacktest() 提交任务，后端返回 commandId。前端随后每 3 秒轮询 api.backtestStatus(commandId:)。页面状态机经历 idle → configuring → running → completed / failed 五个阶段。对比模式下，前端收集选中回测的 runId 列表，调用 api.compareBacktests(runIds:) 获取对比矩阵数据。风险规则在前端本地计算，基于回测结果中的 KPI 与预设阈值比较。模拟运行通过 APIStrategyRuns.listRuns(mode: "dryrun") 获取。
