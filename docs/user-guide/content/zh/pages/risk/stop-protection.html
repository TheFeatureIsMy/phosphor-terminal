止损保护

页面 · 风控 · 5 分钟

止损保护为每个持仓维护一组四层止损价格——从宽松的结构止损到严格的交易所保护止损，逐层收紧确保不会爆仓。

TL;DR

止损保护页面以持仓卡片列表展示每个未平仓位的四层止损：原始结构止损（最宽松）、最后已知好止损、安全运行时止损、交易所保护止损（最严格）。每个卡片左侧有一个价格梯可视化，用彩色标记显示入场价、当前价和各止损层的位置。当波动率异常时，卡片顶部会出现"波动率锁定"徽章，表示止损已冻结不可调整。

谁用 · 何时用

任何持有未平仓位的交易者都应该定期检查止损保护页。每次价格大幅波动后、或者系统推送了"structure_stop_violated"告警时，都该来这里看看止损层是否被突破。这个页面是只读监控页——调整止损参数在策略配置中完成。

页面解剖

  1. ① 状态横幅 — 当系统检测到止损异常时（如交易所止损与计算止损不匹配），显示橙色警示横幅和具体的 reason codes。
  2. ② 持仓卡片 — 每张卡片对应一个未平仓持仓，显示交易对、方向（LONG/绿、SHORT/红）、波动率锁定徽章和刷新/锁定操作按钮。
  3. ③ 价格梯 — 垂直价格标尺，用彩色圆点和横线标记入场价（灰）、当前价（绿）和各层止损的位置。红色止损线距离当前价越近，风险越高。
  4. ④ 止损层级 — 四层止损价格列表，按宽松到严格排列，左侧有色标指示层级。
  5. ⑤ Reason Codes — 每个持仓底部的标签显示该持仓的状态码，如 structure_stop_valid（正常）或 exchange_stop_discrepancy（不一致）。
  6. ⑥ 正常持仓 — 当所有止损都有效时，只显示简洁摘要和 "all stops valid" 绿色徽章。

关键指标怎么读

  • 四层止损价格 — 最宽松的"原始结构止损"基于市场结构（如之前的低点），最严格的"交易所保护止损"是实际发到交易所的止损单。如果当前价格接近交易所保护止损，该持仓风险很高。四层之间的差距越大，说明止损调整空间越大。
  • 波动率锁定 (VOL LOCKED) — 当市场波动率超过阈值时，止损价格被冻结不可调整。这是为了防止在剧烈波动中过早被扫出局。红色脉冲徽章表示该锁定处于激活状态。
  • 交易所止损不一致 (exchange_stop_discrepancy) — 当系统计算的安全止损与实际发到交易所的止损价格差异超过 0.5% 时出现。需要手动检查哪个价格是正确的。
  • 价格梯 — 当前价（绿色圆点）离最近的止损横线越近，该持仓越危险。一个好的持仓应该有足够的"止损空间"让价格正常波动而不触发平仓。

常见操作 · 一步步

  1. 监控所有持仓的止损状态 — 打开止损保护页面，从上到下浏览持仓卡片。注意有黄色或红色 reason codes 的持仓。
  2. 检查波动率锁定 — 查看是否有持仓显示了"VOL LOCKED"徽章。如果有，说明波动率异常期间止损被冻结，不会自动调整。
  3. 刷新止损 — 点击持仓卡片的刷新按钮，系统会重新计算各层止损价格并更新显示。
  4. 强制锁定止损 — 点击锁形按钮强制冻结某个持仓的止损价格（手动模式）。再次点击解除锁定。

幕后

止损保护数据来自 /api/risk/stop-protection。后端 StopProtectionService 先从 Redis 读取缓存的止损价格，然后通过 Freqtrade REST API 获取交易所上实际设置的止损单价格进行比对。如果发现差异超过 0.5%，会在持仓上标记 exchange_stop_discrepancy 并将整体状态升级为 warning。如果 Redis 和 Freqtrade 都不可用，返回模拟数据。价格梯上的"原始结构止损"基于市场结构分析（如最近的 swing low/high），"最后已知好止损"是在结构被破坏前记录的最后一个有效止损。
