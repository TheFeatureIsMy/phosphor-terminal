风控中心

页面 · 风控 · 5 分钟

风控中心是 AlphaLoop 的风险总览仪表盘——用一个弧形仪表盘告诉你系统整体风险处于什么状态，以及哪个守卫快要被触发了。

TL;DR

风控中心用一个大型弧形仪表盘显示账户的整体风险状态（SAFE / WARN / RISK / LOCK / HALT）。下面是四个风控守卫的卡片网格，分别跟踪日亏损、周亏损、总敞口和连续亏损的消耗比例。右下方的紧急操作区提供"禁止新开仓"和"紧急停止"按钮。整个页面背景会根据风险等级改变呼吸光效。

谁用 · 何时用

每天开盘前第一件事——看风控中心一眼就知道系统是否安全。当熔断触发、告警推送或你只是想确认"今天还能不能继续交易"时，风控中心是起点。它不处理具体持仓的止损（那个在止损保护页），而是告诉你整体账户的安全水位。

页面解剖

  1. ① 弧形仪表盘 — 270 度弧线仪表盘，指针位置对应当前风险状态：SAFE（绿）、WARN（黄）、RISK（橙）、LOCK（红）、HALT（深红）。非正常状态下外圈有脉冲光晕。
  2. ② 快速统计 — 系统状态、守卫告警数（告警/总数）、紧急锁定是否激活、风险原因码数量。
  3. ③ 风控守卫 — 每张卡片展示一个风险限制指标的当前消耗比例、数值/上限和状态色码（healthy=绿 / warning=黄 / critical=橙 / breached=红）。
  4. ④ 紧急操作 — "禁止新开仓"阻止新订单但不平现有持仓；"解除禁止"恢复；"紧急停止"发出终止信号并市价平仓（需二次确认）。

关键指标怎么读

  • 风险状态 (SAFE / WARN / RISK / LOCK / HALT) — SAFE 表示所有守卫正常。WARN 表示至少一个守卫接近阈值。RISK 表示多个守卫超阈值。LOCK 表示系统已自动锁定。HALT 表示紧急停止已执行。
  • 守卫消耗百分比 — 如"日亏损 120 / 500 (24%)"，消耗越低越好。超过 80% 进入 warning，超过 100% 进入 breached。单日回撤超过阈值会触发熔断。
  • 紧急锁定 (ACTIVE / OFF) — ACTIVE 表示系统已被锁定，任何新交易和修改操作都被拒绝。需要人工在紧急操作区解锁。
  • Reason Codes — 当系统非正常时，这里显示具体的原因码供排查，例如 daily_loss_limit_reached、exchange_stop_discrepancy。

常见操作 · 一步步

  1. 查看整体风险 — 打开风控中心页面，弧形仪表盘和统计摘要会立即加载。绿色表示一切正常。
  2. 检查守卫详情 — 浏览守卫卡片网格，注意消耗比例接近 80% 以上的卡片。点击卡片可跳转到对应限制的配置页面。
  3. 紧急停止所有交易 — 点击"紧急停止"按钮 → 在弹出的确认对话框中确认 → 系统向所有运行中的策略发送终止信号 → 状态变为 HALT。
  4. 解除禁止 — 在问题解决后，点击"解除禁止"按钮恢复系统正常操作。

幕后

风控中心的数据来自后端 BFF 端点 /api/risk/overview。后端首先通过 RiskAggregator 从 Redis 实时缓存中读取各守卫状态和账户状态。如果 Redis 不可用或数据为空，会回退到数据库查询风控事件表，最后如果数据库也为空则返回模拟示例数据。数据每次打开页面时加载一次，不会自动轮询——需要手动刷新。紧急停止、禁止新开仓和解除禁止分别调用 /api/risk/emergency-stop、/api/risk/block-new-entries 和 /api/risk/unblock 三个 POST 端点。
