订单与持仓

页面 · 执行 · 5 分钟

在这里追踪每一笔订单的生命周期——从委托到成交，以及当前所有持仓的盈亏情况。

TL;DR

订单与持仓页面用两个 Tab 展示数据：订单 Tab 列出所有历史委托（含买卖方向、数量、价格、状态），持仓 Tab 显示当前开仓头寸（含入场价、现价、未实现盈亏、止损价）。顶部有 Exchange 和 Freqtrade 连接状态指示器，异常时显示橙色/红色警告横幅。

谁用 · 何时用

交易员每天查看持仓状态时用这个页面。需要确认某笔委托是否成交、查看当前浮动盈亏、或者检查止损是否触发时，都在这里操作。它也用作执行记录页面的详情入口——点击运行记录中的"查看订单"即可跳转到具体订单列表。

页面解剖

  1. ① 连接状态 — 实时显示交易所和 Freqtrade 的连接健康度。绿色=正常，红色=异常。异常时会展开横幅提示具体 reason_codes。
  2. ② Tab 切换 — 订单 Tab 和持仓 Tab 各带一个计数徽章。选中 Tab 用绿色高亮，未选中用灰色。
  3. ③ 订单/持仓列表 — 每行显示一条记录。订单行显示方向（BUY/SELL）、交易对、订单类型、数量、价格、状态徽章和交易所订单 ID。持仓行额外显示入场价、现价、未实现盈亏和止损价。
  4. ④ 色码说明 — 状态色码和方向色码一目了然。盈亏用绿色（正）和红色（负）区分。

关键指标怎么读

  • 订单状态 — filled（已成交）表示委托已完全执行；pending/open（待成交）表示仍在订单簿上等待；partially_filled（部分成交）需要注意剩余部分是否会继续成交；cancelled（已取消）或 rejected（被拒）需要排查原因。
  • 未实现盈亏 (PnL) — 持仓行显示浮盈/浮亏的绝对值和百分比。正数绿色，负数红色。大幅浮亏时要检查止损是否合理，或者是否应该手动干预。
  • 入场价 vs 现价 — 两者差距直接反映盈亏。如果现价接近止损价，系统会自动触发平仓——但建议你留意预警，不要等到最后才行动。
  • reason_codes — 黄色三角形表示该订单或持仓有附加信息（如风控标记、来源追踪等）。悬停可查看详情。

常见操作 · 一步步

  1. 查看订单详情 — 在执行记录页面点击"查看订单"按钮，会弹出详情弹窗，包含订单列表、账本事件和信号追溯链。
  2. 按模式/状态过滤 — 在执行记录页面使用过滤栏，按 backtest / dryrun / live_small / live 模式或 running / completed / stopped / failed 状态筛选运行记录。
  3. 查看追溯链 — 在详情弹窗的 "Trace" Tab 中，从 Signal → Agent → Strategy → TradeIntent → RiskDecision → Order 六步链路一目了然。
  4. 跟踪账本事件 — 详情弹窗的 "Events" Tab 显示每笔交易的资金变动（trade_pnl / fee / funding / deposit / withdrawal），带时间线和余额变化。

幕后

订单和持仓数据来自后端 BFF 端点 /api/execution/orders-positions。Freqtrade 通过 REST API 获取实时挂单和持仓状态，同时后端直接读取 Freqtrade 的 SQLite 数据库获取历史订单详情。两路数据在 Reconciliation Bus 中对账——如果本地记录与交易所状态不一致，会在 reason_codes 中标记差异。追溯链数据通过 TradeSourceTrace 组件呈现，整合了信号源、策略 DSL 版本、运行时快照和风控决策。
