执行中心

页面 · 执行 · 5 分钟

执行中心是 AlphaLoop 的总控台——从这里你可以看到所有策略会话的运行状态、切换干跑/实盘模式、一键紧急停止。

TL;DR

执行中心汇总了所有正在运行的 Freqtrade 会话。顶部五张卡片分别显示运行中会话数、持仓数、挂单数、Freqtrade 心跳状态和执行延迟。下方是每个会话的详细列表，标注模式（干跑/实盘/纸交易）、策略名和持仓/挂单计数。右上角的红色紧急停止按钮可以在极端情况下立即终止所有交易。

谁用 · 何时用

任何需要知道"当前策略在跑吗？跑得怎么样？"的时候，都该先看执行中心。每天早上开盘前、策略部署后、收到异常告警时，执行中心是你的第一站。如果你只想看一个页面来了解系统是否健康，就看这个。

页面解剖

  1. ① 状态横幅 — 当 Freqtrade 心跳丢失或交易所连接异常时出现，显示具体的 reason_codes 供排查。
  2. ② 汇总卡片 — 运行中会话数、总持仓、总挂单、Freqtrade 心跳（Healthy/Degraded/Down）、执行延迟（<100ms 绿 / <300ms 黄 / 以上橙）。右侧紧急停止按钮需二次确认。
  3. ③ 会话列表 — 每行一个 Freqtrade 会话，显示策略名、交易对、模式徽章（dryrun=黄 / live=红 / paper=绿）、持仓数和挂单数。黄色三角形表示该会话有 reason_codes 需要关注。
  4. ④ 模式色码 — 绿=运行中 / 黄=暂停 / 灰=已停止 / 红=错误。

关键指标怎么读

  • 运行中会话数 — 当前有多少个策略会话在活动。如果比预期少，检查是否有会话意外停止。
  • Freqtrade 心跳 — Healthy 表示 Freqtrade 引擎正常响应；Degraded 表示部分功能可用但延迟偏高；Down 表示无法连接，需要检查 Freqtrade 服务状态。
  • 执行延迟 — 从发起到交易所确认的往返时间。100ms 以内正常，超过 300ms 说明网络或交易所负载异常，可能影响成交质量。
  • 紧急停止 — 点击后向所有会话发送终止信号。只有在系统失控、策略出现异常行为时才使用。使用后需要手动恢复每个会话。

常见操作 · 一步步

  1. 查看会话详情 — 点击任意会话行，跳转到对应的订单/持仓页面查看该会话的完整交易记录。
  2. 紧急停止所有交易 — 点击右上角红色"紧急停止"按钮 → 在弹出的确认对话框中点击"确认停止" → 系统向所有 Freqtrade 会话发送终止信号 → 页面自动刷新显示更新后的状态。
  3. 切换模式 — 执行中心本身不提供模式切换。请在策略管理页面选择 dryrun / paper / live_small / live 模式启动新会话。
  4. 刷新状态 — 页面在打开时自动加载最新数据。如需手动刷新，可切换到其他页面再切回。

幕后

执行中心的数据来自后端 BFF 端点 /api/execution/center。后端首先尝试从 Redis 缓存读取会话状态（实时性要求高），缓存未命中时从数据库查询 Freqtrade 会话表和心跳表，最后如果数据库也空则返回模拟数据。每个会话的状态由 Freqtrade REST API 定期同步（通过 freqtrade_client.py），而历史交易数据则直接读取 Freqtrade 的 SQLite 数据库（freqtrade_db.py）。紧急停止功能调用 /api/execution/emergency-stop 端点，后端向所有运行中的 Freqtrade 会话广播停止命令。
