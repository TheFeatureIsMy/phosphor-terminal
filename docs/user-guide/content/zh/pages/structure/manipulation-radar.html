操纵雷达

页面 · 结构 · 6 分钟

围绕单个疑似操纵案例的九段叙事流。展示多层证据、生命周期阶段、保守/激进两种应对策略与对当前策略的实际影响——并诚实地标注数据缺失和不确定性。

TL;DR

操纵雷达不是定罪系统，而是基于多源数据信号的统计推断引擎。页面顶部是横向滚动的活跃案例条（Hero Strip），点击切换聚焦；下方按九段顺序展开当前聚焦案例的完整证据：判定结论 → 生命周期时间线 → 5 层证据矩阵（价格量能 / 盘口流动性 / 链上 / 社交新闻 / 跨市场）→ 巨鲸与筹码集中 → 跨市场压力 → 社交加速 → 防御与策略联动（保守/激进双栏并列）→ 实时告警 + 相似历史案例。

谁用 · 何时用

所有交易者——尤其是山寨币、低流动性合约的参与者。两类典型使用方式：

  • 谨慎型——在交易前快速核查"这个币是否正在被操纵"。如果显示"疑似派发期"或"疑似崩盘期"，远离。
  • 激进型——理解操纵的生命周期，在建仓/拉升阶段顺势埋伏（系统在防御章节同时给出两种立场的建议供对比）。

页面解剖（九段）

  1. Masthead — 顶部标题 + 系统级免责声明："操纵雷达是统计推断系统，输出基于证据的怀疑而非定罪"。
  2. §0 活跃案例条 Hero Strip — 横向滚动的活跃案例缩略卡。每张卡显示：交易对 · M 类型 · 当前阶段 · 置信度。点击切换下方所有章节的聚焦案例。
  3. §1 判定 Verdict Panel — 当前案例的判定结论：M 类型标签 · 生命周期阶段进度条 · 置信度环 · 风险等级 · 数据完整度（N/5 layers）。所有文案使用概率前缀（"疑似" / "证据指向"），不下定论。
  4. §2 生命周期 Lifecycle Timeline — 水平时间线，5 个阶段节点（SUSPECTED → ACCUMULATE → MARKUP → DISTRIBUTE → COLLAPSE）。当前阶段节点放大发光，已过阶段实心，未到阶段虚线。每个节点可点击展开，看到进入时间 + 当时置信度 + 关键特征快照。
  5. §3 证据矩阵 Evidence Matrix — 5 行 × n 列：Layer A 价格量能 / Layer B 盘口流动性 / Layer C 链上 / Layer D 社交新闻 / Layer E 跨市场。每行显示该层的综合异常分 + data_quality 徽章（数据可靠度 0-1）。data_quality < 0.3 的层标 "Data unavailable" 不参与判定。每行可展开查看具体特征 + z-score + 历史分位。
  6. §4 巨鲸与筹码集中（Layer C 详情放大）— Top-10 持仓集中度变化曲线 + 近 24h 大额转账列表 + 交易所充值净流入。
  7. §5 跨市场压力（Layer E 详情放大）— 资金费率 z-score + 现货-永续基差 + 持仓量变化 + 多空比 + 24h 爆仓金额。
  8. §6 社交加速（Layer D 详情放大，可缺失）— Twitter / Telegram 提及增速 + KOL 提及时间线 + 情绪极端度。若该层数据不可用，整段诚实地显示 "Data unavailable"。
  9. §7 防御与策略联动 Defense & Strategy Impact — 左右双栏并列：保守型建议（蓝色调，例如"立即清仓"）与激进型建议（琥珀色调，例如"减仓或反手做空"）。下方共享区：影响交易对 chip 列表 + 当前策略联动表（每个 enabled 策略显示 ManipulationFilter 是否启用 / 是否会阻断 / reason codes）+ 跳转风控配置按钮。
  10. §8 实时告警 + 相似历史案例 — 左半：该案例的实时告警时间线（阶段切换、异常飙升），通过 WebSocket 推送。右半：基于证据向量相似度匹配的 top-N 历史已完成案例，含 outcome 结局（峰值变化 / 崩盘深度 / 持续天数）作对照。

关键指标怎么读

  • 置信度（confidence） — 0-1 区间。表示证据对该判定的支持强度。受数据完整度限制：缺失越多 layer，置信上限越低（max_confidence = min(completeness × 1.2, 1.0)）。
  • 数据完整度（data completeness） — 当前案例有几层数据可用（满分 5）。3/5 表示只有 3 个 layer 提供了可信数据；判定基于这 3 层，不会用缺失的层"想象"。
  • data_quality 徽章 — 每个 layer 单独标注的数据可靠度。>0.7 可信，0.3-0.7 仅供参考，<0.3 视为不可用。
  • 风险等级 — low / medium / high / extreme，基于综合异常分 + 当前生命周期阶段联合判定。派发期/崩盘期天然上调。
  • 双画像建议 — 保守型给出"远离 / 谨慎参与 / 立即清仓"等防御指引；激进型在同样的证据下给出"小仓埋伏 / 上车 / 反手做空"等顺势建议。两者并列，让用户根据自己的风险偏好与仓位上下文选择立场，而不是系统替你决定。
  • 策略联动 — 列出当前已 enabled 且交易对命中该案例的策略。每条标注其 manipulation_score_filter 规则是否启用、是否会阻断、阻断 reason codes。

常见操作 · 一步步

  1. 打开操纵雷达页面，顶部 Hero Strip 默认聚焦在第一个活跃案例上。
  2. 点击 Strip 中的任意一张卡片切换聚焦——下方所有章节会并行加载新案例的 detail / strategy-impact / similar 三个端点。
  3. 读 §1 Verdict 判定结论，注意置信度环和"N/5 layers"——如果完整度只有 2/5，请把这个判定当作"早期信号"而非"明确结论"。
  4. 跟着 §2 Lifecycle Timeline 理解案例处于哪个阶段。点击节点查看进入时间和当时特征快照。
  5. 在 §3 Evidence Matrix 找到分数最高的 Layer，展开看具体特征（如 volume_zscore=2.4、top_10_concentration=0.62）。
  6. 跳到 §7 Defense & Strategy Impact，对比保守与激进两种立场，决定行动。如果"当前策略联动表"显示某策略的 ManipulationFilter 未启用，点"跳转风控配置"去 Risk Center 检查。
  7. 在 §8 看相似历史案例的 outcome——同类操纵历史上最终崩了多少？持续多久？以此校准心理预期。

幕后

页面数据来自后端 /api/v2/manipulation/* 端点族：/radar（总览 + 活跃案例列表）、/cases/{id}（案例 v2 含分 Layer evidence + data_quality + 双画像信号）、/cases/{id}/strategy-impact（策略联动扫描）、/cases/{id}/similar（相似历史案例）、/alerts（告警轮询）以及 WS /stream（阶段切换 + 新案例实时推送，断线自动退化为 30s polling）。

识别引擎运行三层架构：Layer 1 异常检测器（每层独立计算 z-score + data_quality）→ Layer 2 模式分类器（M1-M8 规则引擎，未来加 ML）→ Layer 3 生命周期追踪器（状态机，suspected→accumulate→markup→distribute→collapse→completed/false_alarm）。案例库在内存中维护（v1），重启会丢失活跃案例；历史案例由 HistoricalManipulationScanner 离线扫描重新发现。
