结构矩阵 · HTF Tribunal

页面 · 结构 · 5 分钟

多周期结构防御的核心战场——用 4×4 矩阵呈现每个交易对在每个时间周期、每种结构区域上的状态，由 8 状态 MTF Guard 状态机给出最终裁决。

TL;DR

结构矩阵（又称 HTF Tribunal）把多周期结构分析做成一个可视化的证据矩阵。行是时间周期（5m / 15m / 1H / 4H），列是结构区域类型（OB / FVG / LP）。每个格子显示该周期该区域的结构强度和状态。矩阵上方有一个倒计时环——显示当前最慢 K 线剩余时间。右侧是裁决面板：允许、观察、要求确认、减仓、或阻止开仓。下方还有影子窗口面板、指控列表和听证时间线。

谁用 · 何时用

管理多条策略、需要多周期一致性确认的量化交易者。适合在信号产生后、执行前打开——让 MTF Guard 对信号涉及的标的进行多周期结构审查，确保入场方向与 HTF 结构一致。也适合在持仓期间定期检查结构是否仍有效。

页面解剖

  1. ① Fast Track Health — 显示数据延迟、数据新鲜度和 Redis 连接状态。绿色表示数据可信，橙色表示数据可能过时。
  2. ② 证据矩阵 — 4 行（时间周期）× 3 列（区域类型）。每个格子显示该周期该区域的强度百分比和状态。绿色 = 结构健康，黄色 = 警示，红色 = 已破坏。带有阴影斜线的格子表示正处于影子窗口中（快周期已侵入慢周期区域）。
  3. ③ 倒计时环 — 显示当前最慢 K 线（如 1H）的剩余收盘时间。环的进度代表蜡烛已走完的比例。时间到意味着 MTF Guard 将做出最终裁决。
  4. ④ 裁决面板 — MTF Guard 的最终决定：Allow（允许交易）、Observe（观察）、Require Confirmation（需要确认）、Reduce Size（减仓）、Block Entry（阻止开仓）。可一键将裁决应用到订单表单。
  5. ⑤ 影子窗口 — 快周期价格进入慢周期结构区域时触发。影子窗口面板显示哪些周期对正在被监控、违规次数和回补进度。

关键指标怎么读

  • 格子颜色 — 绿色（强度 ≥ 70%）表示结构稳固；黄色（40-70%）表示需谨慎；红色（< 40%）表示结构可能已破坏。如果同一列（如 OB）从 4H 到 5m 都是绿色，说明该区域在各周期均被尊重——信号可靠性高。
  • Shadow Window 标记 — 格子上的橙色斜线 + "IN SHADOW" 标签表示快周期价格已侵入慢周期结构区域。这是一个临时状态——慢蜡烛收盘后裁决会确认或驳回。
  • 裁决动词 — Allow = 可以交易；Observe = 等待但暂不行动；Require Confirmation = 需要额外确认才可入场；Reduce Size = 允许交易但减仓；Block Entry = 阻止任何新开仓。
  • 指控列表 — 证据矩阵中每个格子的问题区域被汇总为"指控"，用罗马数字编号。每条指控包含时间周期、区域类型和原因码（如 fast_tf_entered_htf_zone）。

常见操作 · 一步步

  1. 在顶部选择交易对和基准时间周期（通常选策略使用的主周期）。
  2. 检查 Fast Track Health 是否为绿色——数据过期时裁决不可信。
  3. 扫视证据矩阵：你的交易方向是否在各周期得到一致支持？如果 4H 和 1H 是绿色但 5m 是红色，可能只是短线噪音。
  4. 查看裁决面板——如果显示 Block Entry，不要强行开仓。
  5. 如有影子窗口活跃，关注倒计时环。时间到后回来看裁决是否变化。
  6. 点击格子打开详情抽屉，查看该格子的完整数据（强度、填充率、状态、原因码）。

幕后

页面数据通过 5 个并行请求加载：/api/structure/matrix（矩阵数据）、/api/structure/shadow-windows（影子窗口）、/api/structure/mtf-guard（MTF Guard 状态和裁决）、/api/structure/mtf-guard/events（历史事件）、/api/structure/fast-track-health（健康检查）。ViewModel 用 async let 并行获取。数据流遵循 Redis → 服务计算 → mock 回退的三层模式。MTF Guard 后端状态机（MTFTemporalGuardService）管理 8 个状态（Inactive → Watching → Temporary Violation → Reclaim Pending → Pending HTF Close → Confirmed / Invalidated / Expired）和 6 种动作（Allow / Observe / Require Confirm / Block Entry / Reduce Size / Ignore）。影子窗口服务（ShadowWindowService）跟踪快周期内 K 线进度。
