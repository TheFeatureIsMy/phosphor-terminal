代理平台

页面 · AI 研究 · 5 分钟

配置、监控和管理你的 AI 交易代理——每个代理是一个独立的研究分析师，定期扫描市场并产生信号。

TL;DR

代理平台是一个 AI 代理的"控制室"。每个代理是一个可独立配置的分析角色，有自己的 LLM 提供商、研究偏好和输出格式。你可以在这里查看所有代理的运行状态、最近产出信号的统计、以及单个代理的详细表现。代理就像你的研究员团队——你可以给他们分配不同的标的研究任务、调整分析深度、甚至选择不同的 AI 模型后端。

谁用 · 何时用

管理多条策略的高级交易者和量化开发者。当你希望 AI 持续跟踪多个标的、定期产出研究报告和信号时，需要来这里配置代理。也适合在部署新代理后回来检查它是否在正常工作、信号质量如何。

页面解剖

  1. ① 顶部栏 — 页面标题和活跃代理总数。代理数量会随你配置的增加而实时更新。
  2. ② Agent 卡片网格 — 每个代理一张卡片，显示名称、LLM 提供商、运行状态指示灯（绿色=在线、橙色=警告、红色=故障）、本周产生信号数量和平均信号质量分数。点击卡片打开详情面板。
  3. ③ 代理详情面板 — 选中代理后可查看和编辑配置：LLM 提供商、研究偏好（如技术+链上、仅情绪等）、输出语言。下方显示该代理近期产出的信号列表。

关键指标怎么读

  • 平均分数 — 该代理所有信号在信号融合系统中的平均得分。持续高于 70% 说明代理表现好；持续低于 40% 可能需要调整配置或检查数据源。
  • 周信号数 — 信号数量本身不是质量指标——有时一个代理一周只发 2 个高精度信号比发 20 个低质量信号更有价值。结合平均分数看。
  • 状态指示灯 — 绿色 = 正常运行；橙色 = 代理运行中但部分数据源异常；红色 = 代理故障（通常是 LLM API 调用失败），需要检查 API Key 或网络连接。

常见操作 · 一步步

  1. 浏览 Agent 卡片网格，查看所有代理的运行状态和性能概览。
  2. 点击某个代理卡片进入详情面板，查看详细配置和近期信号。
  3. 在详情面板中修改 LLM 提供商（如从 OpenAI 切换到 Anthropic）或调整研究偏好。
  4. 回到网格视图，通过右上角的"新增代理"按钮添加新的 AI 代理。
  5. 配置新代理的名称、研究偏好、输出语言和 LLM 提供商。
  6. 一段时间后回到代理平台，检查代理的信号质量——如果平均分数持续偏低，尝试调整研究偏好或更换 LLM 提供商。

幕后

代理平台通过 tradingagents_adapter.py 管理每个代理的配置和生命周期。每个代理本质上是一个 TradingAgents 实例，配置了独立的 LLM 提供商和偏好参数。代理运行时产生的信号经过 signal_scoring.py 的评分体系（基于可验证性、证据充分性、具体性、新颖性、风险意识五个维度，加权得到 0-5 分）后进入信号中心。代理的配置持久化在数据库中，每次启动时加载。
