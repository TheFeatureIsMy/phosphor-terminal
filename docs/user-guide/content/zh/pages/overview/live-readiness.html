实盘准入控制台

页面 · 总览 · 5 分钟

单屏可判断：5 级总状态 + 11 项门禁 + 三重启动确认。基础设施 / 策略 / 资金 / 风控 / 交易所任一未配，准入即被阻断。

TL;DR

实盘准入是一个一屏可判断的控制台。它汇总 11 项关键门禁（mode / strategy / capital / risk_config / exchange / data_source / validation / backtest / dryrun / notification / emergency_stop），推导 5 级总状态：不可实盘 → 待配置 → 待验证 → 模拟通过 → 可进入实盘。启动实盘必须经过三重确认（启动摘要 → 勾选理解 → 输入确认短语 I confirm live trading），不允许误触发。

谁用 · 何时用

每次从模拟交易切换到实盘之前。新策略上线 / 修改资金池 / 切换交易所后。日常运营中如果 Top Bar 状态条出现异常，也应该来这里看具体哪一项门禁失败了。

页面解剖（一屏布局）

  1. ① Header + 5 级总状态徽章 — 颜色随状态变化：绿(ready_for_live) / 青(paper_passed) / 琥珀(needs_validation) / 黄(needs_config) / 红(not_live)。点击「重新检查」强制 POST `/api/overview/live-readiness/check`。
  2. ② Selection — 4 项选择器 — mode / strategy / capital pool / exchange 各自下拉选择。切换后自动 POST 重算 grand_status。
  3. ③ Gates — 11 项门禁 — 按 group 分两列。左：mode / strategy / capital / risk；右：system / execution。每个 chip 显示「圆点 + label + value」，颜色绿=OK / 橙=WARN / 红=FAIL。
  4. ④ Context — 上下文摘要 — 6 chip 网格（通知 / AI 模型 / 数据源 / 交易所 / freqtrade / redis）+ 2 风险 gauge（日损使用 / 周损使用）+ 紧急停止 chip。全部来自真实端点。
  5. ⑤ Launch — 启动授权 — 阻断项列表（如有）+ 3 个启动按钮（启动模拟 / 启动小仓实盘 / 启动全仓实盘）。按钮按 canStartPaper / canStartLiveSmall / canStartFullLive 启用/禁用。

5 级总状态

  • 不可实盘 (not_live) — 基础设施 / DB / Redis / 数据源不可用。修复后才能继续。
  • 待配置 (needs_config) — mode / strategy / capital / risk / exchange 任一未选择。
  • 待验证 (needs_validation) — DSL 验证 / 回测 / 模拟未通过。
  • 模拟通过 (paper_passed) — 模拟 healthy + mode=paper。
  • 可进入实盘 (ready_for_live) — 全部门禁通过 + mode=live_small/full。

三重启动确认（防误触）

  1. Step 1 阅读摘要 — 弹窗显示 mode / strategy / exchange / capital / grand_status 摘要。
  2. Step 2 勾选确认 — 必须勾选"我已理解"才能进入下一步。
  3. Step 3 输入确认短语 — 必须输入 I confirm live trading 才能点击 LAUNCH。
  4. 真实调用 APIEmergency.emergencyStop + 重新 loadData()。

关键指标怎么读

  • 5 级总状态 — 一眼判断能否进入实盘。颜色 + 徽章 + 描述三重提示。
  • 11 项门禁 — 每项显示「OK / WARN / FAIL」+ 当前值 + 阈值。任一 FAIL = 阻断。
  • 上下文 — 6 chip 让交易员确认系统全局状态（通知 / AI 模型 / 数据源 / 交易所 / freqtrade / redis）。
  • 风险 gauge — 日损 / 周损使用率，红 = 接近熔断。

常见操作 · 一步步

上线前检查

  1. 打开"实盘准入控制台"，查看顶部 5 级总状态徽章颜色（绿 = 可进入实盘）。
  2. 确认 4 个选择器都有选中值（mode / strategy / capital pool / exchange）。
  3. 扫一眼 11 项门禁：任一项 FAIL = 阻断项，定位修复。
  4. 查看上下文 6 chip：全部为绿色 = 平台健康。
  5. 点击"重新检查"按钮触发一次新的健康检查（POST /api/overview/live-readiness/check）。
  6. 点击对应模式的启动按钮（"启动小仓实盘"等），进入三重确认弹窗。

幕后

页面数据通过 GET /api/overview/live-readiness?mode=...&strategy_id=... 端点获取，使用 APIOverview 服务。ViewModel 7 源并行（readiness / notifications / overview / ai_models / risk / capital）。后端综合 11 项 _check_* + _derive_grand_status() 推导 5 级总状态。
