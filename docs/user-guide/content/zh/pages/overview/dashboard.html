AI 总控台

页面 · 总览 · 5 分钟

每天打开 AlphaLoop 第一个看到的页面——AI 对市场的判断、当前持仓风险、需要你确认的事项，全部集中在一屏之内。

TL;DR

AI 总控台是 PulseDesk 的首页。它按优先级分为三层：P0（核心关注）包括 AI 市场判断、持仓与风险状态、待确认事项；P1（分析）包括 Agent 信号分布、策略状态、权益曲线；P2（风控统计）显示风险拦截数据。顶部的工作流导轨展示 Daily Trading Loop 各环节的实时状态，点击即可跳转到对应页面。

谁用 · 何时用

每一位交易者每天早上打开应用后的第一站。在开盘前查看 AI 对市场的判断方向、当前持仓是否安全、有没有需要手动确认的事项。日常操作中，每隔一段时间切回来看一眼，确认一切正常。

页面解剖

  1. ① 交易工作流导轨 — 横向滚动的 Daily Trading Loop 九步流程。每个步骤用不同颜色的霓虹卡片表示，点击即可跳转到对应页面（如信号中心、策略工作区、执行中心）。导轨上方的流动光效显示当前进度，右上角的状态徽章（就绪/需关注/阻塞）一目了然。
  2. ② AI 状态栏 — 四个实时状态指标：AI 服务商（正常/降级/不可用）、本地 GPU（运行中/空闲/不可用）、今日 AI 费用、排队任务数。每 30 秒自动刷新。
  3. ③ AI 市场判断 — AI 综合近期信号后给出的方向判断（看多/看空/震荡），附带置信度进度条、风险等级、来源 Agent 名称以及判断理由。置信度越高、风险越低，信号越可靠。
  4. ④ 持仓 + 风险 — 当前所有开仓头寸，每行显示标的、方向、盈亏（含百分比）、AI 建议（持有/减仓/止盈/平仓）和风险等级（低/中/高）。绿色盈利、红色亏损，一目了然。
  5. ⑤ 待确认事项 — 需要人工干预的项目：待处理的信号、待部署的策略、待释放的风险锁定。每条显示类型图标、标题和描述，右侧有"拒绝"和"批准"按钮。

P1/P2 分析区域默认折叠，点击"展开更多分析"可查看 Agent 信号分布、策略状态总览、权益曲线图和风险拦截统计。

关键指标怎么读

  • AI 市场判断方向 — 显示"看多"、"看空"或"震荡"。方向来自最近 10 个信号的投票——如果多头信号数量超过空头两倍则为看多，反之看空，否则震荡。大数字 + 方向图标让你一眼掌握 AI 态度。
  • 置信度 — 0%~100%，显示在方向下方的进度条里。置信度越高（≥75%），AI 的判断越坚定。低于 50% 时风险等级自动标记为"高"，需要你额外留意。
  • 持仓盈亏 (PnL) — 每个持仓显示绝对盈亏金额和百分比。AI 建议（持有/减仓/止盈/平仓）根据盈亏幅度自动生成：盈利 >5% 建议止盈，亏损 >3% 建议减仓。
  • 风险等级 — 每笔持仓的独立风险标记。盈亏波动 >5% 为高风险，>2% 为中风险。结合 AI 建议一起参考。
  • 待确认数量 — 状态栏右侧的橙色徽章数字。数量为 0 说明没有积压事项；数字越大，需要你花时间处理的事项越多。
  • Agent 信号分布 — 展开后可见。每个 AI Agent 发出的信号总数以及看多/看空的比例。条形越长、绿色占比越高，说明该 Agent 越偏向多头。
  • 风险拦截统计 — 按"已拒绝"、"已减仓"、"仅模拟"、"已放行"四种结果分类的风险事件计数。下方的堆叠条直观显示风控系统的拦截率。

常见操作 · 一步步

处理待确认事项

  1. 在"待确认事项"卡片中找到需要处理的项目，阅读标题和描述了解具体内容。
  2. 点击右侧绿色的"批准"按钮确认操作（例如批准策略部署到下一阶段），或点击"拒绝"取消。
  3. 被处理的项目会立即从列表中消失。注意：批准/拒绝操作目前仅在本地生效，不会影响后端状态。

通过工作流导轨导航

  1. 在页面顶部的"交易工作流"导轨上横向滚动，找到目标步骤卡片。
  2. 查看卡片下方的状态文字（已通过/运行中/需关注/已阻塞），了解该环节当前状况。
  3. 点击任意步骤卡片，自动跳转到对应的功能页面，无需手动从侧边栏寻找。

幕后

Dashboard 是数据加载最密集的页面之一。进入页面时，ViewModel 会并行发起 7 个 API 调用（权益曲线、信号列表、持仓、策略列表、风险事件、系统状态、推理任务队列）。所有请求使用 async let 并发执行，最慢的接口决定总加载时间。进入页面后每 30 秒自动轮询刷新信号、持仓和任务队列数据，离开页面时自动停止轮询。后端不可用时自动降级到模拟数据（Mock），确保 UI 始终有内容可展示。
