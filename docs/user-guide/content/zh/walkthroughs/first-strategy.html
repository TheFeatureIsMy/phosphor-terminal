第一个策略：从零到实盘

情景 · 1 / 5 · 8 分钟

从头脑里的一个交易想法开始，经过 AI 研究、策略创建、回测验证、干跑观察，最终以小仓位上线实盘——完成一条完整的策略生命周期。

开始前 · 检查清单

  • ☐ 后端已启动，Dashboard 正常加载
  • ☐ Freqtrade 服务运行中，交易所 API 已配置
  • ☐ 至少有一个交易对的行情数据可用（推荐 BTC/USDT）

流程图

步骤

1 · 用 AI 投研生成策略想法（AI 投研室）

打开 AI 投研室，输入你想交易的标的（比如 BTC/USDT）。选择 Standard 研究深度，点"开始研究"。系统会并行派出 7 个研究员角色，分别从技术面、情绪面、新闻面、链上分析等角度分析市场。2 分钟后你会看到最终评级面板。

  • 看什么: 最终评级的综合置信度
  • 怎么判断: 置信度 ≥ 70% 且方向明确（看多或看空），说明市场信号足够强，值得据此构建策略。40-70% 需要谨慎，低于 40% 建议换一个标的或等一等。
  • 下一步: 如果评级可用，点击"生成策略草稿"按钮，系统会自动跳转到策略工作区并填入初始参数。

2 · 创建策略（策略工作区）

策略工作区中你会看到一个预填的策略草稿卡片——名称、标的、初始参数已经来自 AI 研究的输出。确认一下策略名称和交易对是否正确。你也可以点击"画布"标签页，进入策略画布，用拖拽节点的方式可视化调整逻辑。

  • 看什么: 策略名称旁边的状态标签——应该是"草稿"
  • 怎么判断: 状态为"草稿"说明策略还未生效。来源如果是"AI Research"说明研究结果已正确传递。右上角应该显示一个绿色"验证通过"或黄色警告。
  • 下一步: 验证通过后，点击"回测"标签页准备验证。

3 · 运行回测（回测与模拟）

在回测标签页，设置时间范围（建议至少 3-6 个月历史数据）、初始资金（默认 100,000 USDT）和交易标的。点击"启动回测"。系统会以异步方式执行——你可以在结果面板看到轮询进度。回测完成后，结果区域会显示总收益率、夏普比率、最大回撤、胜率等关键指标。

  • 看什么: 总收益率、夏普比率和最大回撤
  • 怎么判断: 夏普比率 > 1.5 说明风险调整后收益合理。最大回撤不超过你的心理底线（通常 < 20%）。总收益率跑赢基准即可——不要追求不切实际的超高收益。
  • 下一步: 如果结果满意，切换到"模拟"标签页，启动纸交易。如果结果不理想，回到策略详情页修改参数或 DSL 规则，重新回测。

4 · 启动干跑（执行中心）

回到策略详情页，点"启动模拟"按钮（或在执行中心的操作菜单中选择"启动干跑"）。系统会在 Freqtrade 中以 dry-run 模式运行你的策略——接收实时行情，但不下真实订单。你可以在执行中心看到新创建的会话，模式标注为"dryrun"。

  • 看什么: 执行中心顶部汇总卡片的"运行中"计数是否增加，新会话的状态是否为绿色圆点
  • 怎么判断: 绿色圆点 + "dryrun" 标签 = 正常运行。延迟在 200ms 以内正常。
  • 下一步: 让它跑至少 1 周。这期间每天来执行中心看一眼，确认信号生成和下单逻辑正常。

5 · 实盘准入检查（实盘准入检查）

干跑 1 周后，如果你对策略表现满意，打开实盘准入检查页面。系统会自动检查 Fast Track 延迟、Redis RTT、Freqtrade 状态、交易所 API 权重等各项指标，计算一个 0-100 的准入分数，并告诉你当前允许的交易模式。

  • 看什么: 准入分数和右侧的状态横幅
  • 怎么判断: 准入分数 ≥ 80 且状态为 LIVE_SMALL_READY，表示可以上线小仓。如果有红色阻断项（比如交易所 API 权重耗尽或 Freqtrade 心跳超时），需要先解决它们。
  • 下一步: 准入通过后，回到执行中心。

6 · 切换实盘小仓（执行中心）

在执行中心找到你的策略会话，点击模式标签旁边的操作菜单，选择"切换为实盘"。系统会弹出确认对话框——建议设置一个小仓位比例（比如 10-20% 的资金）。确认后会话模式从 "dryrun" 变为 "live"，你的策略开始用真金白银交易。

  • 看什么: 会话的模式标签从黄色 "dryrun" 变为红色 "live"
  • 怎么判断: 红色 "live" 标签 + 正常的心跳 = 实盘运行中。首次实盘建议前 3 天每几小时来看一次。
  • 下一步: 持续观察至少 1 周。如果表现稳定，可以逐步增加仓位。如果出现异常，随时可以点"紧急停止"。

结束后 · 自验证

  • ☐ 执行中心显示策略会话模式为 "live"，状态绿色
  • ☐ 订单/持仓页面可以看到真实挂单和持仓
  • ☐ Dashboard 的持仓区域显示对应标的的 PnL

遇到问题

  • 策略无法创建 — 检查名称是否重复，或 DSL 规则是否有语法错误。查看策略详情页底部的验证状态栏。
  • 回测一直"排队中" — 检查 Fast Track 服务是否正常运行（实盘准入检查页可以看）。也检查数据源管理页面，确认历史数据已下载。
  • 干跑没有产生任何信号 — 在策略详情页检查信号日志。可能是条件设置太严格。也可以回到 AI 投研室检查当前市场状态是否适合该策略类型。
  • 实盘准入不通过 — 展开警告项列表，逐一解决。最常见的原因是交易所 API 权重不足或 Freqtrade 心跳超时。
  • 切换实盘后立即出现亏损 — 这是正常的滑点效应。小仓位运行几天，如果亏损持续扩大，点"紧急停止"并回到回测页面重新分析。
