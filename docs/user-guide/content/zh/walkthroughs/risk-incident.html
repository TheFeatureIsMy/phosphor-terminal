风险事件处理

情景 · D / 5 · 8 分钟

断路器触发后，从告警到定位原因、暂停策略、复盘回顾，最后决定是否恢复。

开始前 · 检查清单

  • ☐ 已有运行中的策略（干跑或实盘）
  • ☐ 风控系统已开启，至少一个断路器处于激活状态
  • ☐ 已完成 情景 · 第一个策略

流程图

步骤

1 · 收到告警 (仪表盘 / 推送通知)

你的手机或桌面弹出一条推送："策略 BTC-USDT 多头 — 日亏损断路器触发"。你打开 AlphaLoop 仪表盘，看到顶部风险状态栏已经从绿色变为红色，并显示被触发的断路器名称和策略名称。

  • 看什么: 仪表盘顶部风险状态条的颜色和断路器名称。红色 = 有断路器触发，需要立即关注。黄色 = 风控预警（接近阈值但尚未触发）。
  • 怎么判断: 如果是日亏损断路器（Daily DD），意味着策略当日亏损超过了设定的阈值（例如 5%）。如果是连续亏损断路器（Consecutive Losses），意味着策略连续亏损了 N 笔。如果是波动率断路器（Vol Spike），意味着市场波动率突然飙升。
  • 下一步: 点击风险状态条进入风险中心（步骤 2）。如果是误报（如测试环境），可以在告警面板中标记为已读。

2 · 在风险中心定位断路器 (风险中心)

风险中心以卡片形式展示所有已配置的断路器及其当前状态。被触发的断路器卡片会高亮显示，包含触发时间、当前值和阈值。

  • 看什么: 被高亮的断路器卡片——"日亏损 5.8% / 阈值 5.0%"。注意触发时间和当前值与阈值的差距。
  • 怎么判断: 如果超出的幅度很小（5.8% vs 5.0%），可能是正常波动，但规则就是规则——先处理再调整。如果大幅超出（如 12% vs 5%），说明策略可能存在系统性缺陷。
  • 下一步: 点击断路器卡片查看详情（步骤 3）。

3 · 阅读断路器事件详情 (断路器页面)

断路器详情页显示一条完整的事件时间线：触发前最后一笔交易、触发时的账户状态、以及触发后系统自动执行的动作（如停止开仓）。

  • 看什么: 事件时间线中的每一笔交易记录——触发前的最后一笔亏损交易、当时的市场条件（波动率、流动性）、以及触发后系统的响应动作。
  • 怎么判断: 寻找模式——是单笔巨额亏损导致的，还是多笔小额亏损累积的？前者可能只是运气不好，后者可能意味着策略逻辑有问题。注意时间线中是否有"低流动性时段"的标签。
  • 下一步: 了解原因后 → 去执行中心暂停策略（步骤 4）；或者如果确认是偶发事件且风险可控 → 跳过暂停直接恢复交易。

4 · 暂停策略 (执行中心)

执行中心显示所有运行中的策略列表。找到被触发的策略，点击暂停按钮。暂停后策略将不再产生新信号，现有持仓继续持有或按预设规则平仓。

  • 看什么: 策略状态从"运行中"变为"已暂停"。确认没有新的订单被提交。
  • 怎么判断: 暂停不会自动平仓——你可以选择手动平仓（如果市场条件恶化）或让现有持仓自然止盈止损。
  • 下一步: 暂停确认后 → 去成长回顾做复盘（步骤 5）。

5 · 在成长回顾中做事后分析 (成长回顾)

成长回顾以日历视图展示每日盈亏和交易记录。找到触发断路器的那一天，展开详细复盘。关注亏损交易的共同特征——交易时段、市场条件、信号类型。

  • 看什么: 触发日的所有交易记录——入场时间、方向、盈亏、持有时间。
  • 怎么判断: 如果所有亏损交易都发生在凌晨（低流动性时段），可能需要对交易时段做限制。如果亏损集中在某种信号类型（如 FVG 入场），可能需要调整该信号的评分权重。
  • 下一步: 发现问题 → 在 失败聚类 中进一步分析模式（步骤 6）；或者如果确认是偶发事件 → 直接决定恢复策略（步骤 7）。

6 · 失败聚类分析 (失败聚类)

失败聚类自动将亏损交易按多个维度分组——时间段、市场条件、信号类型、持仓时长。每个聚类显示亏损占比和交易笔数，帮助你快速锁定问题根源。

  • 看什么: 亏损占比最高的聚类。例如"凌晨 2-5 时 · 低流动性 · 60% 亏损占比"。
  • 怎么判断: 如果某个聚类的亏损占比显著高于其他（如 60% vs 平均 20%），说明问题可能集中在特定条件下。如果亏损均匀分布，可能是策略整体表现下滑。
  • 下一步: 找到根因 → 调整风控规则或策略参数（步骤 7）；聚类不清晰 → 增加更多数据再观察。

7 · 决定恢复 / 调整 / 终止 (执行中心 / 策略工作区)

基于复盘结论，做出最终决定：恢复策略运行、调整风控阈值或策略参数后恢复、或者直接终止策略。

  • 看什么: 根据分析结果决定行动。如果是阈值过紧导致的误触发，在 断路器 页面调整阈值。如果是策略问题，在 策略工作区 修改参数。
  • 怎么判断: 偶发事件（如黑天鹅波动）→ 恢复交易。参数需要微调 → 调整后恢复。策略逻辑有系统性缺陷 → 终止策略，回到 第一个策略 重新设计。
  • 下一步: 恢复后密切监控 24 小时；终止后归档策略数据供后续参考。

结束后 · 自验证

  • ☐ 策略处于明确状态——运行中、已暂停或已终止
  • ☐ 你记录了触发原因，并能用一句话说清楚"是什么导致了断路器触发"
  • ☐ 风控配置已更新（如果需要）：阈值调整、新增规则或交易时段限制

遇到问题

  • 暂停按钮不可用 — 确认你对该策略有操作权限。如果是共享策略，需要管理员权限。
  • 成长回顾中找不到触发日 — 检查时间范围筛选器是否覆盖了触发日期。默认只显示最近 7 天。
  • 断路器已恢复但策略不开单 — 断路器触发后即使状态恢复，策略也不会自动恢复开仓。需要你手动在 执行中心 点击恢复。
