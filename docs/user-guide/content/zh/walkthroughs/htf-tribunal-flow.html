HTF Tribunal 流程

情景 · C / 5 · 8 分钟

低周期结构突破后，走完结构矩阵 + 影子窗口 + HTF 收盘裁决的全流程，最后把确认的参数送到执行中心下单。

开始前 · 检查清单

  • ☐ 已有运行中的策略，且已产生至少一次信号
  • ☐ 对 SMC/ICT 基础概念（BOS、CHoCH、OB、FVG）有一定了解
  • ☐ 已完成 情景 · 第一个策略 和 情景 · 每日交易循环

流程图

步骤

1 · 信号到达，切到结构矩阵 (策略工作区 → 结构矩阵)

你的策略在 5 分钟图上捕捉到了一个多头信号——价格突破了前一个 HH，BOS 确认。但你不会直接入场，因为低周期的突破可能是假突破。你打开结构矩阵，让 MTF Guard 做多周期审查。

  • 看什么: 快速扫一眼 Fast Track Health 条——必须是绿色，数据过期时裁决不可信。
  • 怎么判断: 绿色表示数据延迟 < 100ms，Redis 连接正常；橙色或红色说明数据可能已过时，先排查数据源。
  • 下一步: Fast Track 健康 → 看证据矩阵（步骤 2）；不健康 → 先查 数据源管理。

2 · 读证据矩阵 (结构矩阵)

4x3 矩阵以颜色直观地告诉你每个周期每个结构区域的状态。你的 BOS 发生在 5 分钟图——检查 4H 和 1H 的 OB 和 FVG 列是否支持这个方向。如果高周期也是绿色，说明结构一致性好。

  • 看什么: 4H 和 1H 的 OB / FVG 列强度百分比。绿色 ≥ 70%，黄色 40-70%，红色 < 40%。
  • 怎么判断: 如果 4H OB 是绿色（强度 85%），1H OB 也是绿色（78%），说明 HTF 结构站在你这边。如果 4H 是红色，说明你的入场方向与大周期趋势相反，风险很高。
  • 下一步: 高周期支持 → 看裁决面板（步骤 3）；高周期不支撑 → 放弃这个信号，返回策略工作区等待。

3 · 影子窗口激活，等待裁决 (结构矩阵)

价格在 5 分钟图上突破了结构，但这个突破发生在 4H OB 区域内——快周期已经侵入了慢周期的结构区域。结构矩阵顶部亮起橙色影子窗口指示条："5m → 1H 结构违规待确认"。倒计时环开始走动。

  • 看什么: 倒计时环上的剩余时间，以及影子窗口面板中的违规计数和回补进度。
  • 怎么判断: 倒计时环显示 1H 蜡烛还有 42 分钟收盘。在收盘之前，裁决面板保持 OBSERVE（观察）状态，不允许入场。如果价格在收盘前回到区域外（回补），影子窗口可能提前关闭。
  • 下一步: 等待倒计时归零（步骤 4）；或者关注回补进度，如果价格快速回到安全区域可以提前刷新矩阵。

4 · HTF 收盘，裁决下达 (结构矩阵)

1H 蜡烛终于收盘了。MTF Guard 状态机根据价格在收盘时的位置做出最终裁决。你刷新结构矩阵，看到裁决面板从 OBSERVE 变成了 ALLOW 或 BLOCK ENTRY。

  • 看什么: 裁决面板的动词——ALLOW（绿色）、OBSERVE（黄色）、REQUIRE CONFIRMATION（黄色）、REDUCE SIZE（橙色）、BLOCK ENTRY（红色）。
  • 怎么判断: ALLOW = HTF 结构确认了低周期的突破，可以入场。BLOCK ENTRY = HTF 结构拒绝了低周期的突破，不要开仓。REQUIRE CONFIRMATION = 还需要额外确认（比如等待下一个 K 线开盘价确认）。
  • 下一步: ALLOW → 步骤 5；BLOCK ENTRY → 放弃信号；REQUIRE CONFIRMATION → 等待额外条件触发后再检查。

5 · 裁决参数复制到执行中心 (结构矩阵 → 执行中心)

裁决面板下方有一个"发送到执行中心"按钮。点击后，裁决中携带的参数（交易对、方向、入场价格区间、止损参考、建议仓位比例）会自动填充到执行中心的订单表单中。

  • 看什么: 订单表单是否已填入正确的参数。特别注意入场价格区间和止损参考是否与结构矩阵一致。
  • 怎么判断: 如果裁决是 REDUCE SIZE，订单表单中的仓位比例应该自动减半。检查仓位比例是否匹配裁决建议。
  • 下一步: 参数正确 → 检查账户状态（步骤 6）；参数异常 → 手动调整或回到结构矩阵重新获取裁决。

6 · 检查账户状态，提交订单 (执行中心)

执行中心右侧显示账户余额、当前持仓和可用保证金。在提交之前，确认你的资金状况可以支持这笔交易，并且没有与现有持仓冲突。

  • 看什么: 可用保证金是否足够；是否有同方向的对冲持仓。
  • 怎么判断: 可用保证金应至少为订单保证金的 150%，留足安全边际。
  • 下一步: 确认 → 点击提交订单；资金不足 → 减仓或跳过。

7 · 在 订单与持仓 中跟踪 (订单与持仓)

订单提交后，切换到订单与持仓页面监控执行状态。挂单会显示为"待成交"，成交后变为持仓。持仓期间建议定期回到结构矩阵检查结构是否仍然有效。

  • 看什么: 订单状态（待成交 / 部分成交 / 完全成交 / 已取消），持仓的浮动盈亏和结构有效性。
  • 怎么判断: 如果持仓期间 MTF Guard 状态从 ALLOW 变为 OBSERVE 或 BLOCK ENTRY，说明结构发生了变化，需要考虑提前离场。
  • 下一步: 持仓持有中 → 加入 每日交易循环 检查清单；已止盈/止损 → 在 成长回顾 中记录。

结束后 · 自验证

  • ☐ 订单已提交，且在订单与持仓页面能看到正确的方向、大小和止损
  • ☐ 你知道当前持仓的结构裁决状态（ALLOW / OBSERVE / BLOCK）
  • ☐ 你理解影子窗口的意义——如果你跳过了步骤 3-4 直接入场，你可能会在一小时后收到一个意料之外的止损

遇到问题

  • 裁决面板一直显示 OBSERVE — 倒计时环可能还在运行，等它归零。如果归零后还是 OBSERVE，可能是影子窗口重新激活了（价格又回到了违规区域）。
  • 参数复制到执行中心后不对 — 检查结构矩阵的交易对选择器是否与策略信号一致。选择了不同的交易对会导致参数不匹配。
  • Fast Track Health 红色 — 数据源可能断开或 Redis 缓存过期。检查 数据源管理 中的数据连接状态。
