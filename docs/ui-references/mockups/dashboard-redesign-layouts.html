RECOMMENDED

A. Bento Command Grid

4列弹性 Bento Grid，信息密度最高。每个 PRD 组件映射到一个固定网格位置，一屏可见全部关键数据。StatusBar 和 EmergencyBar 分别钉顶/钉底。视觉节奏通过卡片大小差异（hero equity 2列 vs 小卡片 1列）创造层次。

优势
  • 信息密度最高，一屏尽览
  • 符合 PRD "总控台"定位
  • Bento 布局天然适配 KryptonCard
  • 最像专业量化交易终端
劣势
  • 窗口过窄时需要响应式降级
  • 卡片内容过多时可能拥挤

方案 B

B. 左右分栏指挥台

左栏 (340px) 纵向堆叠所有状态/指标卡片，右栏展示持仓表格 + 决策流 + 告警时间线。类似 Bloomberg Terminal 的信息面板布局。适合需要长时间盯盘的场景 —— 左侧是"仪表盘"，右侧是"实时流"。

优势
  • 左右分区清晰：指标 vs 事件流
  • 右栏表格/Feed 有更多纵向空间
  • 盯盘模式友好
劣势
  • 左栏窄，EquityCard 展示空间受限
  • 小卡片堆叠可能视觉单调
  • 与 Sidebar 叠加后左侧过重

方案 C

C. 瀑布指令流

单栏纵向瀑布流。顶部大 hero 区（权益 + PnL），中间一行横排 4 个指标卡（Runtime · Readiness · Risk · Emergency），下方全宽持仓表，底部双栏 Decisions + Alerts。最简洁的结构，适合快速扫一眼就走的场景。

优势
  • 布局最简洁，代码量最小
  • Hero 区视觉冲击力强
  • 阅读顺序清晰：从上到下
劣势
  • 需要滚动才能看到底部 Feed
  • 信息密度低于 Bento Grid
  • 不够"专业终端"感
