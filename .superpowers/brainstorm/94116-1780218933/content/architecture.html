四层渲染架构

Canvas Z-Stack 分层
z=4
顶层
Overlay 层 — 交互反馈
框选矩形 连线拖拽预览 对齐辅助线 类型校验指示 吸附点预览 搜索高亮

视口外节点不创建 SwiftUI View（占位点保持选择状态）
使用 GeometryReader 获取实际画布尺寸 → 传入 culler 计算可见集

边线层

• 数据流方向粒子沿路径运动（每秒 N 个粒子）
• 同 Canvas 2D 绘制（批量 stroke 调用优化）
z=1

CanvasBackground — 网格 + 扫描线

数据流

用户操作 → StrategyCanvasTab (手势路由)
  → CanvasViewModel (状态变更 + undo记录 + scheduleSave)
    → graph.nodes/edges (@Observable → SwiftUI 自动刷新)
    → ViewportCuller (过滤可见节点)
    → EdgeRouter (计算可见边的屏幕路径)
    → CanvasEdges / NodeView 渲染

自动保存 → CanvasViewModel.scheduleSave() (3s debounce)
  → GraphSerializer.serialize() → APICanvas.save()
  → 失败时 → CanvasErrorNotifier → Toast/状态栏提示
