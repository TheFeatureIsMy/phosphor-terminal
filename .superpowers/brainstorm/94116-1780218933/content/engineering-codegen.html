1. 工程质量提升

↩ 撤销/重做 Coalescing
问题：拖拽节点时每个 onChanged 事件都记录一次 moveNode，撤销需要按无数次

解决：
• startDrag() 时记录 dragStartPosition（不记 undo）
• endDrag() 时比较最终位置与起始位置
• 有变化才记一条 .moveNode（合并整个拖拽）
• 配置变更用 500ms debounce：
  连续改同一个 key → 只保留最早 old 值 + 最新 new 值
• undo 栈上限 100 条（防止内存膨胀）
⚠ 错误处理 + 用户反馈
CanvasErrorNotifier（新文件）：
• 自动保存失败 → 状态栏 Toast："保存失败，10s 后重试"
• 连续失败 3 次 → 警告图标 + "请检查网络连接"
• 加载失败 → 保持空画布 + "无法加载画布数据" 提示

加载状态：
• CanvasViewModel 新增 isLoading: Bool
• 加载中显示骨架屏（4-6 个节点形状闪烁）
• 加载完成后骨架屏淡出，节点淡入

保存状态指示器：
• 画布底部状态栏："已保存" / "保存中..." / "保存失败"
• 未保存更改显示 "● 未保存"（dot 指示器）
🎭 动画过渡
• 节点添加：scale 0.8→1.0 + opacity 0→1（spring）
• 节点删除：scale 1.0→0.8 + opacity 1→0（easeOut 150ms）
• 配置面板：slide + fade 过渡（保留现有 transition）
• 调色板：slide 过渡（保留现有）
• 连线：opacity 动画（新连线闪烁一次吸引注意）
• 搜索跳转：视口 smooth animation 到目标节点
🧪 测试策略
Swift 侧（新增 CanvasViewModelTests）：
• ViewportCuller 单元测试：100 节点随机分布，验证可见集不超过 viewport 范围
• SnapEngine 单元测试：验证吸附阈值和计算结果
• EdgeValidator 单元测试：类型兼容矩阵、循环检测
• CanvasViewModel 单元测试：undo/redo 行为、copy/paste 行为
• GraphSerializer 往返测试

后端（新增 test_canvas.py）：
• CanvasWorkflow CRUD 端点测试
• 大规模 graph JSON 存储/读取测试（200+ 节点）
